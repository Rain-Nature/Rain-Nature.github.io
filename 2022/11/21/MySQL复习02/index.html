MySQL复习02

MySQL
Publish Date:   2022-11-21

函数与约束

1. 字符串函数

​ MySQL中内置了很多字符串函数，常见的如下表：

函数 功能
concat(S1,S2,…,Sn) 字符串拼接，将(S1,S2,…,Sn)拼接为一个字符串
lower(str) 将字符串str全部转为小写
upper(str) 将字符串str全部转为大写
lpad(str,n,pad) 左填充，用字符串pad对str的左边进行填充，达到n个字符串长度
rpad(str,n,pad) 右填充，用字符串pad对str的右边进行填充，达到n个字符串长度
trim(str) 去掉字符串头部和尾部空格
substring(str,start,len) 返回从字符串str从start位置起的len长度的字符串，索引值是从1开始

​ 注意：str可以传类型是字符串的列名

​ 范例：字符串函数的使用

-- concat
SELECT concat('hello','world');

--lower
SELECT lower('Hello');

--upper
SELECT upper('Hello');

--lpad
SELECT lpad('01',5,'0');

--rpad
SELECT rpad('01',5,'0');

--trim
SELECT trim(' hello world ');

--substring
SELECT substring('12345',1,3); //索引值从1开始

2. 数值函数

​ 常见的数值函数如下表：

函数 功能
ceil(x) 向上取整
floor(x) 向下取整
mod(x,y) 返回x/y的模
rand() 返回0~1内的随机数
round(x,y) 得参数x的整体进行四舍五入的值，保留y位小数

​ 范例：数值函数的使用

-- ceil
SELECT ceil(1.1);  //结果2

-- floor
SELSECT floor(1.1) //结果1

-- mod
SELSET mod(3,4);  //3除4的余数，结果3

--rand
SELECT rand(); //0~1随机数

--round
SELECT round(2.345,2); //结果2.35

​ 注意：生成的数字可以作为字符串参与上面字符串函数中

3. 日期函数

​ 常见日期函数如下：

函数 功能
curdate() 返回当前日期
curtime() 返回当前时间
now() 返回当前日期和时间，即返回的是date类型
year(date) 获取指定date的年份
month(date) 获取指定date的月份
day(date) 获取指定date的日期
date_add(date,interval expr type) 在指定日期下增加一个时间间隔，返回该时间值
datediff(date1,date2) 返回起始时间date1和结束时间date2之间的天数，结果为date1-date2

​ 范例：日期函数的使用

-- curdate
SELECT curdate();  //结果2022-11-14

-- curtime
SELECT curtime();  //结果9:29:15

-- now
SELECT now();  //结果2022-11-14 9:29:54

-- year
SELECT year(now()); //结果2022

-- month
SELECT month(now());  //结果11

-- day
SELECT day(now());  //结果14

--date_add
SELECT date_add(now(),INTERVAL 10 DAY);  //结果2022-11-24 10:05:31

-- datediff
SELECT datediff('2022-12-01','2022-11-1'); 结果30，注意如果前小后大则值为负数

4. 流程函数

​ 流程函数在SQL语句中实现条件筛选，从而提高语句的效率

函数 功能
if(value,t,f) 如果value为true，则返回true否则返回f
ifnull(value1,value2) 判定第一个参数是否为空，即如果value1不为空，返回value1，否则返回value2
case [表中字段] when [val] then [res1]…else [default] end 如果val1与表中数据判断为true返回res1,…(可以写多个判断的)否则返回default默认值
case when [表中字段]比较运算 then [res1]…else [default] end 如果不是具体的而是范围，使用该语法

​ 范例：流程控制函数的使用

-- if
SELECT if(true,'ok','error');  //结果ok

--ifnull
SELECT ifnull(null,'error');  //结果error

-- case when then else end
-- 需求：查询员工表emp中员工姓名和工作地址(北京/上海--->一线城市，其他--->二线城市)
SELECT
	name,
	(case workaddress when '北京' then '一线城市' when '上海' then '一线城市' else '二线城市' end) AS '工作地址'
FROM emp;

5. 约束

​ 约束是作用于表中字段上的规则，用于限制存储在表中的数据

​ 约束的目的是保证数据库中数据的正确、有效性和完整性

​ 约束的分类

约束 描述 关键字
非空约束 限制该字段的数据不能为null NOT NULL
唯一约束 保证该字段的所有数据都是唯一、不重复的 UNIQUE
主键约束 主键是一行数据的唯一标识，要求非空且唯一 PRIMARY KEY
默认约束 保存数据时，如果未指定该字段的值，则采用默认值 DEFAULT
检查约束 保证字段值满足某一条件 CHECK
外键约束 用来让两张表的数据之间建立连接，保证数据的一致性和完整性 FOREIGN KEY

​ 注意：约束是作用于表中字段上的，可以在创建表/修改表的时候添加约束

​ 范例：约束的使用

CREATE TABLE user(
	id INT PRIMARY KEY;
	name VARCHAR(20) NOT NULL UNIQUE;
	age INT CHECK(age>0 && age<=120);
	statues CHAR(1) DEFAULT '1';
)COMMENT '用户表'

6.外键约束

​ 概念：用来让两张表的数据之间建立连接，保证数据的一致性和完整性

​ 一般把设立外键的表叫做子表，而让该外键为主键的表叫主表

​ 设置外键后，当要删除主表的数据时，要判定该数据在子表中是否存在关联关系，如果存在则拒绝删除操作

​ 外键约束的语法：

  • 添加外键

    //建表时直接创建
CREATE TABLE 表名(
	字段名 数据类型
	...
	[CONSTRAINT] [外键名称] FOREIGN KEY(外键字段名) REFRENCES 主表(主表列名)
);

//表建立后修改外键
ALTER TABLE 表名 ADD CONSTRAINT 外键名称 FOREIGN KEY(外键字段名) REFERENCES 主表名(主表列名);

  • 删除外键

    ALTER TABLE 表名 DROP FOREIGN KEY 外键名称;

​ 范例：外键的操作

//增加外键
ALTER TABLE emp ADD CONSTRAINT fk_emp_dept_id FOREIGN KEY (dept_id) REFERENCES dept(id);

//删除外键
ALTER TABLE emp DROP FOREIGN KEY fk_emp_dept_id;

​ 外键约束：删除和更新行为

行为 说明
NO ACTION 当主表中删除/更新对应记录时，首先检查该记录是否有对应外键，如果有则不允许删除/更新，外键约束的默认行为
RESTRICT 与NO ACTION一致
CASCADE(级联) 当在主表中删除/更新对应记录时，首先检查该记录是否有对应外键，如果有，则也删除/更新外键在从表中的记录
SET NULL 当在主表中删除对应记录时，首先检查该记录是否对应外键，如果有则设置从表中该外键值为null(此时需要设计时外键允许取null)
SET DEFAULT 主表有变更时，子表将外键列设置成一个默认的值(Innodb不支持)

​ 语法：

ALTER TABLE 表名 ADD CONSTRAINT 外键名称 FOREIGN KEY(外键字段名) REFERENCES 主表名(主表列名) ON UPDATE CASCADE ON DELETE CASCADE

​ ON UPDATE：表示更新时怎么操作

​ ON DELETE：表示删除时怎么操作

​ 前面一段与外键设置时语法相同
