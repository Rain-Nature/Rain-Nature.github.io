JS高级01-JS的面向对象

JS高级
Publish Date:   2024-12-22

1. 作用域

​ 规定变量被访问的范围

​ JS中共有4种作用域：全局作用域、局部作用域(块作用域)、函数作用域(普通函数)以及eval作用域，作用域会影响this的指向

(1)局部作用域：{1.函数作用域、2.块作用域}

​ 1.函数内部声明的变量只能在函数内部被访问，函数执行完毕，函数内部的变量实际被清空了

​ 2.{}包裹的为代码块（包括if的，for的），let、const会产生块作用域，var不会产生块作用域

​ 注意：局部作用域外部未再次定义就使用会报错

(2)全局作用域

​ <script> 标签内和.js文件内，即【最外层】就是所谓的全局作用域，全局作用域中声明的变量，任何其他作用域都可以被访问

(3)垃圾回收机制

  • 引用计数法：跟踪被引用的次数，如果为0则释放内存，缺点：相互引用的变量无法销毁会造成内存泄漏
  • 标记清除法：从根部扫描对象，查不到的对象会被回收，解决了引用计数法的缺点

(4)闭包

​ 内层函数访问到外层函数的作用域：闭包=内层函数 + 外层函数的变量

​

​ 此时，调用fun()函数可以打印出a的值，调用outer()也可以

闭包的应用

​ 1. 在内定义数据，实现数据的私有

闭包引发的问题

​ 因为函数一直被调用，变量无法回收，会导致内存的泄露

(5)变量提升

​ 属于JS语言缺陷，它允许变量声明之前即被访问（仅存在于var声明变量），只提升声明不提升赋值且仅提升到当前作用域最前面，实际开发少用var

2. 函数进阶

(1)函数提升

​ 函数在声明之前就可以被调用，原因：JS会把所有函数声明提升到当前作用域的最前面

<script>
	fn()
	function fu(){
		console('函数提升')
	}
</script>
//函数可以正常执行

(2)普通函数的两种定义格式(命名函数和匿名函数)

命名函数

​ 有一个确切的名字，可以直接使用

​ 语法：

function fn1() { 
	console.log("这是命名函数") 
}
匿名函数

​ 匿名函数本身没有名字，但可以通过变量 fn2 来调用它

​ 匿名函数属于函数表达式，不存在函数提升

​ 语法：

const fn2 = function() { 
	console.log("我是匿名函数表达式") 
}

(3)普通函数(命名函数和匿名函数)的this

​ 命名函数 和 匿名函数在 this 的绑定行为上是相同的，this 的值由函数的调用方式决定

(4)箭头函数

​ 箭头函数属于表达式函数，不存在函数提升

​ 基本语法，有简写小心区分

​ 1. 最基本的语法

<script>
	//箭头函数，参数x
	const fn = (x) => {
		console.log(x)
	}
	//调用
	fn(1)
</script>

​ 2. 当参数只有一个时可以省略小括号，只有一行代码时可以省略大括号

const fn = x => console.log(x)

​ 3. 只有一行代码时可以省略return

const fn = x => x+x

​ 4. 箭头函数可以直接返回一个对象

const fn = (uname) => ({name:uname})
//调用
fn('刘就成')

(5)箭头函数的this（一定要注意作用域）

​ 在普通函数中，每一个新函数根据它是如何被调用的来定义这个函数的this值

​ 箭头函数不会创建自己的this，它只会从自己的作用域链的上一层沿用this，即看上一层作用域的this指向谁

​ 易错代码1

<script>
	const obj = {
  		uname: 'hello',
  		sayHi: () => {
    		console.log(this);  // this指向window，而不是obj，因为obj不是作用域
  		}
	}
	obj.sayHi()
</script>

​ 易错代码2

const obj = {
  name: 'Alice',
  greet: function() {
    // 这里的 this 指向 obj
    setTimeout(() => {
      console.log(this.name); // 箭头函数继承外部 greet() 的 this
    }, 1000);
  }
};

obj.greet();  // 输出 'Alice'

3. 解构、遍历与过滤

​ 解构是快速批量的将[值/对象属性和方法]给变量的简洁语法

(1)数组解构

​ 变量的顺序对应数组单元值的位置

//1.普通情况
const arr = [1,2,3]
//数组解构
const [a,b,c] = arr
console.log(a)  //1
console.log(b)  //2
console.log(c)  //3

//2.变量少，单元值多，可以用剩余参数接
const [a,b,...c] = [1,2,3,4]
console.log(a)  //1
console,log(b)  //2
console.log(c)  //[3,4]

//3.多维数组
const arr = [1,2,[3,4]]
const [a,b,[c,d]] = arr
console.log(a)  //1
console,log(b)  //2
console.log(c)  //3
console.log(d)  //4

(2)对象解构

​ 解构的变量名必须和原对象的属性名一致，如果找不到对应的会被赋值为undefined，要起别名可以使用名称:新名称

//普通情况
const obj = {
	uname:'刘就成',
	age:24
}
const {uname:name,age} = obj 
console.log(name)  //刘就成
console.log(age)  //24

//多级对象解构
const pig = {
	name:'佩奇',
	family:{
		mother:妈妈,
		father:爸爸
	}
}
const {name, family:{mother,father}} = pig
console.log(name)  //佩奇
console.log(mother)  //妈妈
console.log(father)  //爸爸

//可以只要对象中的一部分,在函数传参时可以传pig，写函数体时只用family
const {family} = pig

(3)foreach遍历数组

​ 只能遍历数组不能遍历对象，且内部没有return

基本语法

​ 语法：当前数组元素必写，索引号可选

被遍历的数组.forEach(function(当前数组元素,【当前数组索引号】)){
	//函数体	
})

​ 样例：

const arr = ['ren','green','pink']
arr.forEach(function(item,index){
	console.log(item)   //数组元素：red green pink
	console.log(index)  //索引号：0,1,2
})

//改为箭头函数同时省略index的写法
arr.forEach(item => {
	console.log(item)   //数组元素：red green pink
})

(4)filter方法过滤数组

​ 返回一个符合条件的新数组，过滤不会影响原数组

基本语法

​ 语法：currentValue必须写，index可选

被遍历的数组.filter(function(currentValue,index){
	retun 筛选条件
})

​ 样例：

const arr = [10,20,30]
const newArr = arr.filter(function(item,index){
	return item >= 20
})
console.log(newArr)  //[20,30]

//改为箭头函数同时省略index的写法
const newArr = arr.filter(item => item>= 20)
console.log(newArr)  //[20,30]

4. 深入对象

​ 对象的定义可以用修饰符const、let、var，但是对象里面的属性和方法不可以用修饰符，只能直接通过对象引用来修改它们

(1)创建对象的三种方式

  • 利用对象字面量创建对象

    const o = {
	name:'佩奇'
}

  • 利用new Object创建对象

    const o = new Object({name:'佩奇'})
console.log(o)  //{name:'佩奇'}

//也可以直接往对象变量里追加
const o = new Object()
o.age = 10
console.log(o)  //{age:10}

  • 利用构造函数创建对象

(2)构造函数

​ 构造函数是一种特殊的函数，主要用来创建对象，通过构造函数可以快速创建多个类似的对象

构造函数的约定

​ 1. 构造函数的命名以大写字母开头

​ 2. 构造函数只能由“new”操作符来执行

语法

​ 例子：创建一个猪对象

function Pig(uname,age){
	this.uname = uname
	this.age = age
}
const peiqi = new Pig('佩奇',6)  //实例化
构造函数的存储

​ 函数对象（包括构造函数）是存储在内存中的一个对象，这通常是在堆中，因为函数对象可以是动态创建的，并且可能会在多个地方被引用

​ 如下代码：Person 函数本身是一个对象，它会被存储在堆内存中，通过 new 关键字创建的 person1 实例（对象）也是存储在堆内存中

function Person(name) {
  this.name = name;
}

const person1 = new Person('Alice');

(3)对象的this和普通函数的this以及箭头函数的this

  • 如果你调用一个函数时使用 new 关键字，JavaScript 会将 this 绑定到新创建的对象。
  • 如果函数是作为方法调用的，this 会指向调用该方法的对象。
  • 如果函数是直接调用的，this 会指向全局对象

​ 需要注意的是：箭头函数的 this继承自外部函数或上下文的 this，它没有自己的 this

(4)实例成员和静态成员

​ 在 JavaScript 中，属性/方法不一定要在类或构造函数定义时就明确地定义，也可以在实例化对象后通过实例来添加和调用属性/方法

实例成员

​ 通过构造函数创建的对象称为实例对象，实例对象中的属性和方法称为实例成员（实例属性和实例方法）

​ 样例：

//实例成员
function Pig(name){
	//实例属性
	this.name = name
}
const peiqi = new Pig('佩奇')
//实例属性
peiqi.name = '小猪'
//实例方法
peiqi.sayHi = () => {
	console.log('你好')
}
静态成员

​ 构造函数的属性和方法被称为静态成员(静态属性、静态方法)

​ 说明：

 1. 静态成员只能构造函数访问
 2. 静态方法中的this指向构造函数

​ 样例：

function Pig(name){
	//实例属性
	this.name = name
}
//静态属性
Pig.eyes = 2
console.log(Pig.eyes)
//静态方法
Pig.sayHi = function(){
	console.log('hi')
}
Pig.sayHi()

5. 内置构造函数

​ 内置构造函数是 JavaScript 内部预定义的函数，允许你创建特定类型的对象，并提供了一些基本功能和属性

(1)JS的包装类型

​ JS基本数据类型：字符串、数值、布尔、undefined、null

​ 只有对象才有方法，但是在js中定义字符串、数值、布尔的变量也可以调用写好的方法，是因为js底层做了封装，将这些基本数据类型转为了复杂数据类型(对象)

(2)其他内置构造函数

Object

​ 定义：

let obj = new Object();
// 或者简写为：
let obj = {};

​ Object常用的静态方法

  • Object.keys(对象名)：获取对象中所有的属性，返回的是一个数组

    Object.keys(Pig)

  • Object.values(对象名)：获取对象所有属性的值，返回也是一个数组

    Object.values(Pig)

​

Array

​ 数组的核心方法如下图所示：传参都为回调函数，reduce多一个初始值

​ map是对数组所有元素进行相同处理

​

​ 样例：(forEach，filter，map，reduce)

const numbers = [1, 2, 3, 4, 5];

// forEach:输出数组的每个元素
numbers.forEach(function(num) {
  console.log(num);
});

// filter:过滤出大于 3 的数字
const filtered = numbers.filter(function(num) {
  return num > 3;
});
console.log(filtered);  // [4, 5]

// map:每个数字乘以 2
const doubled = numbers.map(function(num) {
  return num * 2;
});
console.log(doubled);  // [2, 4, 6, 8, 10]

// reduce:计算数组所有元素的总和，初始值是0，可以省略但会以数组的第一个值为初始值所以最好不要省
//accumulator是计数值(初始值由函数第二个参数决定)，currentValue是元素值(如果是对象就是完整对象)，计算时需要把数据拆出来
const sum = numbers.reduce(function(accumulator, currentValue) {
  return accumulator + currentValue;
}, 0);
console.log(sum);  // 15
