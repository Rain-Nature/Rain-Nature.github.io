​ 注意1：主键使用id标签，其他字段使用result标签

​ 注意2：在 MyBatis 的 resultMap 标签中，type 属性用于指定映射结果的类型。它指定了结果映射的目标 Java 类型，表示将查询结果映射到哪个 Java 对象。图中因为起了别名所以没有使用全限定名

​ 在 MyBatis 中，resultMap 和 resultType 都是用于映射查询结果的标签，但它们在使用和功能上有一些区别。

  1. resultMap：resultMap 是 MyBatis 中定义映射关系的标签，通过 resultMap 可以定义如何将查询结果映射到 Java 对象的属性。resultMap 可以包含多个 result 子标签，每个 result 标签指定了一个查询结果字段与 Java 对象属性之间的映射关系。使用 resultMap 可以方便地处理复杂的映射关系，如一对一、一对多等关联关系。resultMap 的定义可以在 XML 配置文件中，也可以通过注解方式定义。
  2. resultType：resultType 的映射与 Java 实体类是相关的。当使用 resultType 时，MyBatis 会将查询结果自动映射到指定的 Java 类型对象上。resultType 是 MyBatis 中直接指定查询结果类型的标签，通过 resultType 可以指定返回结果的 Java 类型。resultType 可以是任何可被映射的 Java 类型，如基本类型（int、String等）、自定义类、Map等。在使用 resultType 的情况下，MyBatis 会通过自动映射将查询结果的列名与对应的 Java 对象属性名进行匹配，并将结果赋值给对应的属性。相比于 resultMap，resultType 适用于简单的查询映射，不涉及复杂的关联关系。

6. 多表查询知识小结与补充

​ 补充

​ 在 MyBatis 中，<collection> 和 <association> 元素都用于处理对象之间的关联关系，但它们有一些关键的区别。

  1. 处理的数据类型：
    • <association> 用于处理一对一的关联关系，其中一个对象包含对另一个对象的引用。
    • <collection> 用于处理一对多的关联关系，其中一个对象包含多个关联对象的集合。
  2. 结果映射方式：
    • <association> 通过指定关联对象的属性和映射规则来创建关联对象，并将其设置到父对象中的对应属性中。
    • <collection> 通过指定关联对象集合的属性和映射规则，以及用于构建每个关联对象的结果映射，来创建关联对象集合，并将其设置到父对象中的对应属性中。
  3. 子元素的定义：
    • <association> 元素可以包含 <id> 和 <result> 子元素来映射关联对象的属性。
    • <collection> 元素可以包含 <id>、<result>、<association>、<collection> 等子元素来映射关联对象的属性以及关联对象集合中的元素。
