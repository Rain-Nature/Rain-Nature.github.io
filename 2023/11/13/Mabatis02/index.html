​ 上述为最基本的使用代理的查询方式，但是注意，查询到的数据有些是封装上的，有些没有被封装上为null。原因是因为表的字段名称与实体类的字段名称不一致，如表中叫brand_name，而实体类中叫brandName

​ 解决方案1：将查询的sql语句起别名，别名与实体类一致；缺点每个sql查询都要定义一次别名

​ 解决方案1优化：使用sql片段防止重复写一段sql，注意sql标签要写在mapper标签里面；缺点不灵活

<!--在BrandMapper.xml中使用sql片段-->
<mapper namespace="mapper.BrandMapper">
    
	<!--sql片段-->
    <sql id="brand_column">
        id,brand_name as brandName,company_name as companyName,ordered,description,status
    </sql>
    
    <select id="selectAll" resultType="pojo.Brand">
        select <include refid="brand_column"/>
        from tb_brand
    </select>
</mapper>

​ 解决方案2：使用resultMap灵活的解决（最常用）

​ resultMap可以将实体类属性与表的字段映射，使用resultMap替换之前的resultType

<!-- 自己对比代码感受一下，首先是type="pojo.Brand"是全限定名，id是唯一标识-->
<!-- 这里<select id="selectAll" resultMap="brandResultMap">使用的是resultMap，而不是resultType-->
<mapper namespace="mapper.BrandMapper">
    <!-- property是实体类属性名，column是数据库里列名-->
    <resultMap id="brandResultMap" type="pojo.Brand">
        <!-- 主键用id，其他键用result-->
        <result property="brandName" column="brand_name"></result>
        <result property="companyName" column="company_name"></result>
    </resultMap>
    
    <select id="selectAll" resultMap="brandResultMap">
        select *
        from tb_brand
    </select>
</mapper>

​ 知识点：mybatis参数占位符

  1. #{}：在执行sql时会将其替换为?，可以防止SQL注入

    select * from tb_brand where id=#{id}
//假设传值1
//执行结果：select * from tb_brand where id=?

  2. ${}：拼sql，会存在SQL注入问题

    select * from tb_brand where id=${id}
//假设传值1
//执行结果：select * from tb_brand where id=1

  3. 使用时机

    • 参数传递的时候使用：#{}
    • 动态传递sql的表名或列名不固定时使用：${}

​ 知识点：特殊字符处理方法

​ 因为sql语句是写在xml文件中，所以有些字符不能正常使用比如<

  1. 使用转义字符

    //原sql：select * from tb_brand where id<${id}
//转义改为
select * from tb_brand where id &lt; ${id}

  2. 使用CDATA区

    //原sql：select * from tb_brand where id<${id}
//CDATA区<![CDATA[ 内容 ]]>，中间写要用的字符
select * from tb_brandwhere id <![CDATA[ < ]]> #{id}
(3)多条件查询(难点)：通过多个条件查询记录

​ 对于多条件参数的接收mybatis有三种接收方式

  1. 使用注解@Param接收：

    List<Brand> selectByCondition(@Param("status")int status,@Param("companyName")String companyName,@Param("brandName")String brandName);

    注意：@Param(“值”)要和xml中#{值}相对应

  2. 使用实体类封装后传参：用实体类set方法封装

    List<Brand> selectByCondition(Brand brand);

    注意：这里要作为条件的brand实体类的属性要和xml中#{}里的值一一对应，不然会封装不上

  3. 使用map封装后传参：

    List<Brand> selectByCondition(Map map);

    注意：这里要作为条件的Map的键要和xml中#{}里的值一一对应，不然会封装不上

​

​ 上面三个条件查询mapper统一对应的xml如下所示

<select id="selectByCondition" resultMap="brandResultMap">
	select *
	from tb_brand
	where
		status=#{status}
		and company_name like #{companyName}
		and brand_name like #{brandName}
</select>

3. 动态sql语句

​ MyBatis的动态sql提供if、choose、trim、foreach

(1) 动态sql-if

​ 其中where标签类似于1=1，让sql语句在没有where条件的时候也能正常执行，但1=1在没有条件时会查询出表中所有字段，而使用where标签就不会

​ if标签中的test必写，test里面写判断条件，写的是传入参数的判断，标签里写要拼凑的sql语句

(2) 动态sql-choose(when,otherwise)：选择一个，类似Java中的switch语句

​ 注意：这里where语句可以使用<where> 标签

(3) 动态sql-foreach

​ 对于where传递参数像in关键字(多个or)时，可以使用foreach

​ 注意，collection 属性的值是传递集合应该与要传递的集合的名字一致，该集合可以是Java自带的，也可以是自定义的，以便 MyBatis 可以正确地将参数值传递给 foreach 元素进行循环迭代。

​ open 属性指定在循环开始时要输出的字符串，separator 属性指定在每个元素之间要输出的字符串，close 属性指定在循环结束时要输出的字符串。

​ foreach 元素的 item 属性指定循环中的当前元素变量名，其类似于增强for循环的x，作循环输出使用。#{id} 是一个占位符，代表当前循环的元素，这里使用了 MyBatis 的参数注入方式。

​ item="id" 表示在循环中每个元素的变量名是 id。#{id} 则用于在 SQL 语句中引用这个变量，并将它替换为实际的元素值
