Spring03-监听器存储spring容器与SpringMVC的使用

SSM
Publish Date:   2023-04-25

1. Spring容器对象的管理

问题：

​ Spring容器对象需要通过new ClasspathXmlApplicationContext(Spring的xml配置文件)或new AnnotationConfigApplicationContext(SpringConfiguration.class)获得，但在编写Web项目，不同的类中，每当需要获得Bean时都要编写一次，这样会使得Spring容器对象创建多次

(1)Web项目：使用自定义监听器管理Spring容器对象(非重点用于引出Spring自带的监听器)

​ 在Web项目中，可以使用ServletContextListener监听整个Web应用启动和销毁的生命周期，当Web应用启动时，就可创建Spring容器对象，将其存储在最大的域servletContext中，这样在任意位置都可以从域中获得Spring容器对象

步骤

​ 1.在监听器的包内定义一个ServletContextListener类

​ 2.在web.xml将这个ServletContextListener类进行配置

​ 3.其他Servlet、Filter、Listener类通过this.getServletContext来获得存储在servletContext域中的Spring容器对象

样例代码

​ 1.在listener包下新建ContextLoaderListener.java，继承实现servletContextListener接口

//ContextLoaderListener.java

public class ContextLoaderListener implements ServletContextListener{
	//上下文初始化时调用的方法
	public void contextInitialized(ServletContextEvent servletContextEvent){
		ApplicationContext app = new ClassPathXmlApplicationContext("applicationContext.xml");
		//将app存储到最大的域（ServletContext）中
		ServletContext servletContext = servletContextEvent.getServletContext();
		servletContext.setAttribute("app",app);
	}
}

​ 2.在web.xml文件，配置ContextLoaderListener.java，告知tomcat有这个监听器

//web.xml

<web-app version="3.0" ...>
	//配置自定义监听器
	<listener>
		<listener-class>com.itheima.listener.ContextLoaderListener</listener-class>
	</listener>

​ 3.Web层代码-UserServlet类中，通过this.getServletContext来获得存储在servletContext域中的Spring容器对象

//UserServlet.java

public class UserServlet extends HttpServlet{
	
	@Override
	protected void doGet(HttpServletRequest req,HttpServletResponse resp) throws Exception{
		//获取上下文
		ServletContext servletContext = this.getServletContext();
		//获取在servletContext里的Spring容器对象
		ApplicationContext app = (ApplicationContext)servletContext.getAttribute("app");
		//使用Spring容器对象，获得容器里的bean
		UserService userService = app.getBean(UserService.class);
	}
}

(2)Web项目：使用Spring封装的监听器管理Spring容器对象（重点）

​ Spring提供了监听器ContextLoaderListener，其内部基于ServletContextListener进行封装，其还提供了工具类WebApplicationContextUtils用于获得Spring容器对象，在Web项目中的主要作用是：1.加载Spring容器对象，2.将该容器封装在Web的上下文对象中

步骤

​ 1.在pom.xml文件导入spring-web坐标

​ 2.在web.xml中配置ContextLoaderListener监听器，以及导入Spring配置文件applicationContext.xml

​ 3.使用WebApplicationContextUtils获得Spring容器对象

样例代码

​ 1.在pom.xml文件导入spring-web坐标

//pom.xml

<dependency>
	<groupId>org.springframework</groupId>
	<artifactId>>spring-web</artifactId>
	<version>5.0.5</version>
</dependency>

​ 2.在web.xml文件下配置Spring提供的监听器，并在配置前告知监听器加载Spring配置文件的路径，监听器会通过这个路径的文件自动创建Spring容器对象（该功能是Spring封装好的）

//web.xml

<web-app version="3.0" ...>
	//告知监听器加载Spring配置文件的路径，监听器会通过这个文件获得Spring容器对象
	<context-param>
		<param-name>contextConfigLocation</param-name>
		<param-value>classpath:applicationContext.xml</param-value>
	</context-param>
	
	//配置Spring的监听器
	<listener>
		<listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>
	</listener>

​ 3.在UserServlet.java，编写获得Web上下文，再通过Web上下文获得Spring容器对象，再通过Spring容器对象获得bean

//UserServlet.java

public class UserServlet extends HttpServlet{
	
	@Override
	protected void doGet(HttpServletRequest req,HttpServletResponse resp) throws Exception{
		//获取上下文
		ServletContext servletContext = this.getServletContext();
		//获取在servletContext里的Spring容器对象
		WebApplicationContext app = WebApplicationContextUtils.getWebApplicationContext(servletContext);
		//使用Spring容器对象，获得容器里的bean
		UserService userService = app.getBean(UserService.class);
	}
}

2. SpringMVC框架的概述

​ Spring MVC 基于Servlet，其通常作为Web层的一部分，它可以独立于其他Spring模块来运行，也可以结合Spring的其他功能，如Spring事务管理、数据访问、AOP等

​ SpringMVC可以通过一套注解让简单的Java类成为处理请求的控制器（servlet类），而无需实现任何接口，同时还支持RESTful编程风格的请求

(1)SpringMVC框架的使用

​ Spring MVC 项目的设计上有一个 上下文分层 的概念，Web 上下文和 Root 上下文（Spring容器对象）是分开的。这种分层有助于提高模块化和可维护性。applicationContext.xml 通常用于定义非 Web 相关的服务和数据访问对象，而 spring-mvc.xml 负责配置 Web 相关的组件（例如，处理请求的控制器、视图解析器等）

​ 当 DispatcherServlet 加载 spring-mvc.xml 时，它会创建一个 Web Application Context，用于管理 Web 层相关的 Bean

开发步骤

​ 1.导入SpringMVC坐标
​ 2.在web.xml中配置SpringMVC前端控制器，并配置加载spring-mvc.xml
​ 3.新建spring-mvc.xml作为SpringMVC的配置文件，在该文件中配置Spring组件扫描扫描web层的包，配置SpringMVC注解的启用
​ 4.编写UserController.java，用注解将其配置到Spring容器中
​ 5.运行后用浏览器访问资源测试

样例代码

​ 1.在pom.xml文件导入SpringMVC包

//pom.xml

<dependency>
	<groupId>org.springframework</groupId>
	<artifactId>spring-webmvc</artifactId>
	<version>5.0.5.RELEASE</version>
</dependency>

​ 2.在web.xml中配置SpringMVC的前端控制器 ，在 Spring MVC 项目中，spring-mvc.xml 是由 DispatcherServlet 来加载的，而不是 ContextLoaderListener。DispatcherServlet 是 Spring MVC 的核心，它充当前端控制器（Front Controller），负责处理所有 HTTP 请求。DispatcherServlet 会读取它自己的配置文件（通常是 spring-mvc.xml），初始化 Web 层相关的组件，例如控制器、视图解析器、请求映射等

//web.xml

<servlet>
	<servlet-name>DispatcherServlet</servlet-name>
	<servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
	//init-param：用于告知前端控制器配置这个Servlet时配置文件在哪
	<init-param>
		<param-name>contextConfigLocation</param-name>
		<param-value>classpath:spring-mvc.xml</param-value>
	</init-param>
	//load-on-startup:代表服务器启动时就去加载servlet(优先级设置为1)
	<load-on-startup>1</load-on-startup>
</servlet>
//servlet-mapping:为映射地址
<servlet-mapping>
	<servlet-name>DispatcherServlet</servlet-name>
	//这个/表示缺省的servlet，即每次访问任何请求都要走这个servlet，若为YYY.xxx为扩展名为YYY.xxx时才走这个servlet
	<url-pattern>/</url-pattern>
</servlet-mapping>

​ 3.创建SpringMVC的xml文件，进行Spring组件扫描，按定义SpringMVC只管Web层所以只扫描controller，同时启用SpringMVC的注解

//spring-mvc.xml

//需要在命名空间里补一个context的命名空间
<beans xmlns:context="http://www.springframework.org/schema/context"
	   xsi:schemaLocation="http://www.springframework.org/schema/context" http://www.springframework.org/schema/context/spring-context>
	 //配置扫描controller层
	 <context:component-scan base-package="com.itheima.controller"/>
	 //启用Spring MVC的注解，和Spring组件扫描类似
	 <mvc:annotation-driven />

​ 4.编写UserController.java，用注解将其配置到Spring容器中

//UserController.java

@Controller
public class UserController{
	//这个属于SpringMVC的注解，详细功能见下面
	@RequestMapping("/quick")
	public String save(){
		System.out.println("运行成功");
		return "success.jsp";
	}
}

(2)SpringMVC的注解：@RequestMapping(“请求路径”)

​ @RequestMapping是 Spring MVC 中用于处理 HTTP 请求的注解。它可以用于类级别或者方法级别，用来定义请求的 URL 映射、HTTP 方法（如 GET、POST 等）以及请求参数等信息

  • 方法级别：用来映射具体的请求路径，并指定该方法处理哪些 HTTP 请求。
  • 类级别：用来为控制器类定义一个基础的路径，方法级别的路径会在类级别路径的基础上追加。
样例：方法级别

​ 在浏览器请求路径：http://localhost:8080/quick可以访问success.jsp的资源

//UserController.java

@Controller
public class UserController{
	//请求地址为 http://localhost:8080/quick
	@RequestMapping("/quick")
	public String save(){
		System.out.println("运行成功");
		return "success.jsp";
	}
}
样例：类级别

​ 在浏览器请求路径：http://localhost:8080/user/quick可以访问success.jsp的资源

//UserController.java

@Controller
@RequestMapping("/user")
public class UserController{
	//请求地址为 http://localhost:8080/user/quick
	@RequestMapping("/quick")
	public String save(){
		System.out.println("运行成功");
		return "success.jsp";
	}
}
请求路径的参数配置

​ 通过请求路径的参数配置可以进一步细分请求的匹配条件

  • value：用于指定请求的URL，在只有value的情况下可以省略，但多个属性不能省，上面代码完整的应该为@RequestMapping(value="/quick")
  • method：用于指定请求的方式，默认是GET，如果要改为POST，它存储方式是枚举类型为@RequestMapping(value="/quick",method=RequestMethod.POST)

(3)Spring新注解-代替 @RequestMapping

​ Spring 4.3 引入了几个更为具体的注解，用来代替 @RequestMapping，使代码更加语义化

  • @GetMapping：用于处理 GET 请求
  • @PostMapping：用于处理 POST 请求
  • @PutMapping：用于处理 PUT 请求
  • @DeleteMapping：用于处理 DELETE 请求
  • @PatchMapping：用于处理 PATCH 请求

(4)SpringMVC的执行流程

图示

上图流程解释（重点）

​ 当你在浏览器中输入一个 URL（比如 http://localhost:8080/quick）并按下回车时，浏览器会向服务器发起一个 HTTP 请求。这个请求可能包含一些数据（如表单数据、请求头等）。服务器接收到请求后，会做出一个 HTTP 响应，将结果返回给浏览器，通常是网页内容、图片等。Servlet 容器（如 Tomcat）会在每次接收到请求时自动创建这两个对象【HttpServletRequest（简称 req）和 HttpServletResponse（简称 resp）】并将它们传递给 DispatcherServlet（Spring 的前端控制器），DispatcherServlet 将请求路由到对应的控制器方法并将 req和resp对象传递给控制器方法，然后控制器方法就可以使用它们来处理请求和构建响应，最后，控制器方法返回响应的视图或数据，DispatcherServlet会通过 resp将数据发送回浏览器

什么是servlet

​ Servlet是 Java 提供的一种技术，用于在服务器端处理 HTTP 请求和生成 HTTP 响应。简单来说，Servlet 就是一个可以处理 Web 请求的 Java 类。当用户访问一个 Web 应用时，服务器会调用对应的 Servlet 来处理请求。

3. SpringMVC的其他组件

​ 除了DispatcherServlet（前端控制器），要实现请求的处理和响应，Spring MVC 的其他组件还包括 HandlerMapping（请求映射）、Controller（请求处理）、HandlerAdapter（调用控制器）、ModelAndView（模型视图）、ViewResolver（视图解析器）等，这些其他组件都是由DispatcherServlet调用完成工作

​ 这里所有组件的配置都是在spring-web.xml文件中配置

(1)SpringMVC各组件执行流程

图示

SpringMVC代码层面的执行流程（各组件）

​ 1.web.xml加载：在Servlet容器启动时，web.xml配置文件加载加载，配置 DispatcherServlet 和 ContextLoaderListener
​ 2.ContextLoaderListener加载：ContextLoaderListener初始化并加载 Root Web Context（例如 applicationContext.xml），通常包含服务层和数据源的配置
​ 3.DispatcherServlet 加载：DispatcherServlet 加载并初始化 Web ApplicationContext（spring-mvc.xml）
​ 4.请求到达 DispatcherServlet：请求由 DispatcherServlet 拦截，HandlerMapping 查找匹配的控制器（Controller）
​ 5.HandlerAdapter 执行控制器：一旦找到匹配的控制器，DispatcherServlet使用HandlerAdapter来调用控制器方法。HandlerAdapter负责执行控制器方法，并处理控制器方法的返回值（比如视图名或数据）
​ 6.视图解析与响应：DispatcherServlet 通过视图解析器（ViewResolver）渲染视图，将最终响应通过 HttpServletResponse返回给浏览器

(2)SpringMVC组件在spring-mvc.xml的配置

​ SpringMVC组件在原生环境是配置好的，可以在spring-mvc.xml对组件再配置进行功能上增强（aop思想）

样例：视图解析器（viewResolver）的配置

​ 在进行视图跳转时，对每一次请求u进行前缀+u+后缀的拼接作为请求路径

//spring-mvc.xml


//配置视图解析器
<bean id="viewResolver" class="org.springframework.web.servlet.view.InternalResourceViewResolver">
	//用其父类提供的set方法进行注入
	//设置视图路径的前缀
	<property name="prefix" value="/jsp/">
	//设置视图路径的后缀
	<property name="suffix" value=".jsp">
</bean>
