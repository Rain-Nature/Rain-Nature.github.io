Spring01-Bean与IoC

SSM
Publish Date:   2023-04-25

1. 传统Javaweb开发的困惑与解决方案

问题1：传统javaweb的开发中对于数据的操作，往往都需要new对象来完成操作

​ 解决思路：程序代码中不去new对象，第三方根据要求为程序提供需要的Bean对象

问题2：事物功能、日志功能耦合在业务代码中

​ 解决思路：第三方提供Bean对象的**代理对象(BeanProxy)**，该对象是对Bean的增强，可以完成事物操作

2. Spring程序开发步骤

​ 在jsp/servlet中，会用接口来引用接口实现的对象，来实现开发
​
​ 对于Spring开发，会在xml配置文件中用id标识来绑定接口实现类全限定名，而后在service类通过Spring框架读取xml文件根据标识来获取Dao对象，如下图

Spring开发步骤总结：

​ 1.通过maven导入Spring开发的基本坐标
​ 2.编写Dao接口和实现类
​ 3.创建Spring核心配置文件，在main/resources/创建xml文件
​ 4.在xml文件中配置UserDaoImpl
​ 5.在service层使用Spring的API获得Bean实例(Dao)

3. Spring开发的基本坐标

​ 导入的基本坐标如下：

<dependencies>
        <dependency>
            <groupId>org.springframework</groupId>
            <artifactId>spring-context</artifactId>
            <version>5.0.5.RELEASE</version>
        </dependency>
</dependencies>

4. Spring中的Bean

​ 可以把Bean看作是Spring框架中的“对象”或“组件”。简单来说，它就是在Spring容器中由Spring管理的一个类的实例。Spring框架会创建这个类的对象，并在适当的时候将它交给你使用

5. Spring配置文件xml详解

​ 在Spring的XML配置中，XML文件实际上是代替了手动使用new关键字来创建对象

模板如下：

​ 需要使用Spring的xml进行生成xml文件才能获得模板

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">
    
    <bean id="唯一标识" class="类的全限定名"></bean>    

</beans>

(1)Bean标签的最简配置

<bean id="唯一标识" class="类的全限定名"></bean> //class是bean的基本属性

​ 默认情况下它调用的是类中的无参构造函数，如果没有无参构造函数则不能创建成功

(2)Bean标签的范围配置

​ 属性scope：指对象的作用范围，取值如下：

属性值 说明
singleton 默认值，单例
prototype 多例
request WEB项目中，Spring创建一个Bean的对象，将对象存入到request域中
session WEB项目中，Spring创建一个Bean的对象，将对象存入到session域中
global session WEB项目中，应用在Portlet环境，如果没有Portlet环境那么global session相当于session
单例和多例的区别：

​ 单例是不管getBean(“id”)获取几次，都是同一地址的对象，多例获取几次，就创造几个对象，详细如下图所示

(3)与Bean生命周期有关的方法

​ 属性init-method：指定类的初始化时执行的方法
​ 属性destroy-method：指定类的销毁时执行的方法
​
​ 这些要指定的方法需要在接口实现类中编写，在使用bean标签时指定

<bean id="唯一标识" class="类的全限定名" init-method="bean初始化要执行的方法名" destroy-method="bean销毁时要执行的方法名"></bean>

​ bean的创建是在main函数中用Spring的API创建

(4)Bean的实例化(如何创建)三种方法

通过无参构造方法实例化

​ Spring默认使用无参构造器来实例化Bean，Spring容器会自动通过new操作符创建一个Bean对象，然后填充它的属性

//MyBean.java
public class MyBean {
    public MyBean() {
        System.out.println("MyBean constructor");
    }
}

//applicationContext.xml
<bean id="myBean" class="com.example.MyBean"/>
通过(工厂)静态方法实例化：需要使用属性factory-method

​ 工厂方法是在类中定义的一个静态方法或实例方法，用于返回该类的Bean对象

​ factory-method属性用于指向静态工厂方法或指向实例方法来创建Bean，该方法将由Spring容器调用来实例化Bean

//MyBean.java
public class MyBean {
    private MyBean() {
        System.out.println("MyBean constructor");
    }

    public static MyBean createInstance() {
        return new MyBean();
    }
}

//applicationContext.xml
<bean id="myBean" class="com.example.MyBean" factory-method="createInstance"/>
通过工厂类实例化：需要使用属性factory-bean、factory-method

​ factory-bean：通过一个工厂Bean来实例化Bean，工厂Bean中的方法负责创建Bean实例

//MyBean.java
public class MyBean {
    public MyBean() {
        System.out.println("MyBean constructor");
    }
}

//MyBeanFactory.java
public class MyBeanFactory {
    public MyBean createMyBean() {
        return new MyBean();
    }
}

//applicationContext.xml
<bean id="myBeanFactory" class="com.example.MyBeanFactory"/>
<bean id="myBean" factory-bean="myBeanFactory" factory-method="createMyBean"/>

​ 当前重点还是深入了解无参构造方法实例化

6. Bean的依赖注入(DI)：用于实现控制反转（IoC）

​ 依赖注入是指在对象的创建和管理过程中，Spring容器会自动将需要的依赖（其他Bean）注入到当前Bean中，而不需要手动创建这些依赖对象，依赖主要是当前对象需要调用依赖对象提供的方法

(1)非依赖注入：需要的依赖从spring容器外部获取

​ 该方式也使用了spring容器，但与传统只是把new对象变为了对象由spring管理，代码如下：

//Dao层
public class UserDaoImpl implements UserDao{
	public void save(){
		System.out.println("save running...")
	}
}

//service层
public class UserServiceImpl implements UserService{
	public void save(){
		ApplicationContext app = new ClassPathXmlApplicationContext("applicationContext.xml");
		//外部获取spring管理的容器
		UserDao userDao = (UserDao) app.getBean("userDao");
		userDao.save();
	}
}

//controller层
public class UserController{
	public static void main(String[] args){
		ApplicationContext app = new ClassPathXmlApplicationContext("applicationContext.xml");
		UserService userService = (UserService)app.getBean("userService");
		userService.save();
	}
}

//applicationContext.xml
<bean id="userDao" class="com.itheima.dao.impl.UserDaoImpl"></bean>
<bean id="userService" class="com.itheima.service.impl.UserServiceImpl"></bean>

(2)依赖注入：需要的依赖从spring容器内部注入(两种注入方法)

​ 依赖注入是Spring框架核心IoC的具体实现

第一种：使用set方法注入（最常用）
步骤

​ 1.创建DAO或其他Bean，需要被注入到Service层

​ 2.在Service层提供一个set方法，用于接收外部Bean的注入

​ 3.在Controller层调用Service层的方法

详细代码

​ 与非依赖注入的代码相比改的都是service层和xml文件

//Dao层
public class UserDaoImpl implements UserDao{
	public void save(){
		System.out.println("save running...")
	}
}

//service层
public class UserServiceImpl implements UserService{
	private UserDao userDao;
	
	//该方法名用于注入时使用
	public void setUserDao(UserDao userDao){
		this.userDao = userDao;
	}

	public void save(){
		userDao.save();
	}
}

//controller层
public class UserController{
	public static void main(String[] args){
		ApplicationContext app = new ClassPathXmlApplicationContext("applicationContext.xml");
		UserService userService = (UserService)app.getBean("userService");
		userService.save();
	}
}

//applicationContext.xml
<bean id="userDao" class="com.itheima.dao.impl.UserDaoImpl"></bean>
<bean id="userService" class="com.itheima.service.impl.UserServiceImpl">
	/**
		name的值是上面setUserDao去掉set且第一个字母变小写
		ref用于引用对象，这里引用的时userDao的唯一标识id
	*/
	<property name="userDao" ref="userDao"></property>
</bean>
第二种：使用有参构造方法注入：
//Dao层
public class UserDaoImpl implements UserDao{
	public void save(){
		System.out.println("save running...")
	}
}

//service层
public class UserServiceImpl implements UserService{
	private UserDao userDao;
	
	//构造方法实现
	public UserServiceImpl(UserDao userDao){
		this.userDao = userDao;
	}
	
	//补充无参构造
	public UserServiceImpl(){
	}

	public void save(){
		userDao.save();
	}
}

//controller层
public class UserController{
	public static void main(String[] args){
		ApplicationContext app = new ClassPathXmlApplicationContext("applicationContext.xml");
		UserService userService = (UserService)app.getBean("userService");
		userService.save();
	}
}

//applicationContext.xml
<bean id="userDao" class="com.itheima.dao.impl.UserDaoImpl"></bean>
<bean id="userService" class="com.itheima.service.impl.UserServiceImpl">
	/**
		name的值是上面UserServiceImpl构造方法的形参名
		ref用于引用对象，这里引用的时userDao的唯一标识id
	*/
	<constructor-arg name="userDao" ref="userDao"></constructor-arg>
</bean>

7. Bean依赖注入的数据类型

​ 在spring中，除对象的引用可以注入，普通数据类型、集合都可以在容器中进行注入(用set或者构造方法)

(1)普通数据类型的注入（set方式）

样例：String、int的注入
//Dao层
public class UserDaoImpl implements UserDao{
	private String username;
	private int age;
	
	public void setUsername(String username){
		this.username = username;
	}
	
	public void setAge(int age){
		this.age = age;
	}
	
	public void printUser(){
		System.out.println(username);
		System.out.println(age);
	}

}

//applicationContext.xml
<bean id="userDao" class="com.itheima.dao.impl.UserDaoImpl">
	<property name="username" value="张三"/>
	<property name="age" value="18"/>
</bean>

(2)集合的注入（set方式）

样例：List、map、properties的注入
//Dao层
public class UserDaoImpl implements UserDao{
	private List<String> strList;
	private Map<String,User> userMap;
	private Properties properties;
	
	public void setStrList(List<String> strList){
		this.strList = strList;
	}
	
	public void setUserMap(Map<String,User> userMap){
		this.userMap = userMap;
	}
	
	public void setProperties(Properties properties){
		this.properties = properties;
	}
	
	public void printAll(){
		System.out.println(strList);
		System.out.println(userMap);
		System.out.println(properties);
	}
}

//applicationContext.xml
<bean id="userDao" class="com.itheima.dao.impl.UserDaoImpl">
	<property name="strList">
		<List>
			<value>hello</value>
			<value>world</value>
		</List>
	</property>
	<property name="userMap">
		<map>
			//key随便指定，ref注入的对象一定要在容器中
			<entry key="u1" value-ref="user1"></entry>
		</map>
	</property>
	<property name="properties">
		<props>
			//key随便指定
			<prop key="p1">ppp1</prop>
			<prop key="p2">ppp2</prop>
		</props>
	</property>
</bean>
//用于演示ref引用使用
<bean id="user1" class="com.itheima.domain.User">

8. Spring中xml文件的分模块开发

​ 项目中的不同xml模块需要拆分，此时要在主配置文件引入这些模块，需要使用import标签进行加载

基本语法：
<import resource="applicationContext-xxx.xml"/>

​ resource属性值为要引入的Spring配置文件的位置

9. Spring相关API

(1)ApplicationContext接口与它的实现类：用于获得Spring容器对象

​ applicationContext：接口类型，可以通过其来接Spring容器对象

​

ApplicationContext的实现类：获取Spring容器对象的三种方式

​ 1.ClassPathXmlApplicationContext(“参数为resource文件夹的xml文件路径”)：它是从类的根路径下加载配置文件（常用）

​ 2.FileSystemXmlApplicationContext(“参数为xml文件在电脑中的磁盘路径”)：它是从磁盘路径上加载配置文件，配置文件可以在磁盘的任意位置

​ 3.AnnotationConfigApplicationContext：当使用注解配置容器对象时，需要使用此类来创建spring容器，它用来读取注解

(2)getBean()方法

​ getBean()有两个常用的API，一个是按id在spring容器中找，还要一个是按类型在spring容器中找

​ 官方文档见下图：

第一个API：

app.getBean(“id”)

​ 传递的参数是xml文件中设置的id值，其允许spring容器中出现相同类型的bean因为可以用id进行区分

第二个API

app.getBean(Class)

​ 传递的参数是字节码对象类型(xxxx.class)，其只允许spring容器中有一种类型的bean
