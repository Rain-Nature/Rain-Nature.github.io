Spring04-SpringMVC对http请求的接收以及响应的发送

SSM
Publish Date:   2023-04-25

1. SpringMVC注解功能的启用

​ 如果要使用SpringMVC相关的注解，需要在spring-mvc.xml中进行配置，不写的话无法正常使用SpringMVC提供的注解

​ 若是SpringBoot的项目默认自动启用

样例：SpringMVC注解的启用
//spring-mvc.xml

//启用 Spring MVC 注解支持，和Spring注解开发的组件扫描类似
<mvc:annotation-driven />

2. SpringMVC与Http协议的关联

​ SpringMVC 是服务器端的框架，在 SpringMVC 中，请求（Request）是指由客户端发起的 HTTP 请求，响应（Response）是服务器对请求的回应

(1)HttpServletRequest和HttpServletResponse在SpringMVC的使用

​ 在 SpringMVC 中，HttpServletRequest 和 HttpServletResponse 是处理 HTTP 请求和响应的核心对象，通常用于控制器(Controller)的方法中接收和处理 HTTP 请求，以及构建和发送 HTTP 响应。请求和响应分别代表了 HTTP 通信的两个重要阶段。请求阶段是从客户端到服务器的过程，响应阶段是从服务器到客户端的过程

样例：HttpServletRequest 和 HttpServletResponse的使用

​ 写在形参，SpringMVC会自动注入这两个对象

////UserController.java

@RequestMapping("/quick")
public String quick(HttpServletRequest req, HttpServletResponse resp) {
    // 这里的 req 和 resp 可以用来操作请求和响应
}
访问时的过程

​ 1.请求：当你访问 http://localhost:8080/quick?name=lxs 时，浏览器会向服务器发送一个 HTTP 请求。这个请求包含了 URL 和查询参数（例如 name=lxs），HttpServletRequest会接收这个请求，并提供你访问请求参数（如 req.getParameter(“name”)）的方式

​ 2.处理：控制器方法 handleRequest()会根据请求数据进行处理。例如，它从 req 获取查询参数 name，然后可能进行一些业务逻辑处理

​ 3.响应：控制器方法可以通过 HttpServletResponse设置响应的内容。你可以设置响应头、状态码，或者通过 req.setAttribute() 把数据存储到请求对象中（这是功能重叠的地方），并返回视图名

​ 4.视图渲染：SpringMVC 会根据返回的视图名（例如 “lxsPage”）来渲染对应的视图（比如 JSP 页面）。在该页面中，你可以访问之前存储在请求中的数据（如 ${message}）

(2)HttpServletRequest对象

​ 表示客户端发送的 HTTP 请求。它包含了请求的各种信息，例如请求的 URL、请求参数、请求头、请求方法（如 GET 或 POST）、请求体等。通过 HttpServletRequest对象，你可以获取客户端提交的请求参数、请求头信息，甚至是请求体数据；还可以通过 req.setAttribute()存储数据，这些数据会在请求生命周期内有效，且可以通过请求转发传递给视图

req的常用方法
  • req.getParameter(“paramName”)：获取请求中的参数
  • req.getHeader(“headerName”)：获取请求中的头部信息
  • req.getMethod()：获取请求的方法类型（如 GET 或 POST）
HttpServletRequest的生命周期

​ 从 客户端发起请求 时开始，服务器接收到请求后，创建 HttpServletRequest 对象，并将其传递给 SpringMVC 控制器进行处理。在控制器处理请求并返回响应视图时，HttpServletRequest 对象是有效的。当请求完成并返回视图或重定向时，HttpServletRequest 对象的生命周期结束

(3HttpServletResponse对象

​ 表示 HTTP 响应。它是用来向客户端返回响应的对象，可以用来设置响应的状态码、响应头以及响应体的内容；还可以控制页面跳转，有重定向：response.sendRedirect() 会让浏览器发起一个新的请求，响应对象会将浏览器重定向到新的 URL，还有请求转发：RequestDispatcher.forward()用于将请求转发到另一个资源（比如 JSP 页面）并继续在当前请求生命周期内处理。

resp的常用方法
  • resp.setStatus(200)：设置响应状态码为 200（表示成功）
  • resp.setHeader(“Content-Type”, “application/json”)：设置响应头
  • resp.getWriter().write(“response data”)：服务器端向客户端发送响应数据，这个数据需要客户端用axios或fetch来接收
HttpServletResponse的生命周期

​ 从 服务器开始处理请求 并准备响应时开始，HttpServletResponse 用于设置响应头、状态码和响应内容。当服务器准备好响应内容并将其发送给客户端后，HttpServletResponse 的生命周期结束

3. 浏览器对于跳转还是数据的区分

响应方式 状态码 Location 头 响应体
返回数据 200 OK ❌ 无 ✅ 有数据（JSON、HTML、文本等）
跳转 301/302 ✅ 有跳转 URL ❌ 通常没有（有也会被忽略）

4. SpringMVC的数据响应

​ Spring MVC 的数据响应方式包括页面跳转（返回视图）、请求转发、重定向、JSON/XML 数据返回、文件下载、流数据响应、WebSocket 通信、自定义响应实体和异常处理，每种方式根据不同的需求返回不同类型的数据或资源

(1)页面跳转-根据请求路径跳转

​ 最常见的方式，用于传统的 Web 应用，通常配合 JSP来渲染 HTML 页面，跳转的页面一定要有

样例

​ 浏览器访问：http://localhost:8080/quick/success.jsp，会跳转显示success.jsp页面

//UserController.java

@Controller
public class UserController{
	@RequestMapping("/quick")
	public String save(){
		System.out.println("运行成功");
		//默认是转发方式，要重定向需要写为redirect:success.jsp
		return "success.jsp";
	}
}

(2)页面跳转-返回ModelAndView对象（了解）

​ Model表示模型，作用为封装数据；View表示视图，作用为展示页面。如果在返回视图给页面时，只有View，没有Model，那么这种页面跳转方式和直接根据请求路径跳转没有区别

样例

​ 浏览器访问：http://localhost:8080/quick/success.jsp，会跳转显示success.jsp页面

//UserController.java

@Controller
public class UserController{
	@RequestMapping("/quick")
	public String quick(){
		//创建ModelAndView对象，也可以通过形参quick(ModelAndView modelAndView)直接传递，SpringMVC会自动创建该对象
		ModelAndView modelAndView = new ModelAndView();
		//设置模型数据,addObject()第一个参数作为key，第二个参数为value，在jsp页面<h1>标签中用${username}，获取到itcast
		modelAndView.addObject("username","itcast");
		//设置视图名称，访问的时候是success.jsp
		mdoelAndView.setViewName("success");
		return modelAndView;
	}
}

(3)SpringMVC注解-@ResponseBody

​ @ResponseBody：该注解写在方法上，作用是告诉 Spring 框架，方法的返回值应该直接写入到 HTTP 响应体中，而不是放入视图或者模型中。该注解常用在构建 RESTful Web 服务时直接返回数据（比如 JSON 或 XML 格式）

样例：直接返回字符串，不进行跳转显示其他页面

//UserController.java

@Controller
public class UserController{
	@RequestMapping("/quick")
	@ResponseBody
	public String quick(){
		/*
			等价于手动操作HttpServletResponse对象
			response.setContentType("text/plain");
    		response.getWriter().write("你好，世界！");
		*/
		return "你好，世界！";
	}
}

(4)回写数据-手动封装Json数据

步骤

​ 1.导入Json相关的坐标，用于处理对象转Json

​ 2.编写controller逻辑

样例：将对象信息以Json格式回写

​ 1.首先在pom.xml导json相关的坐标，用于将一个对象的数据转化为Json字符串

//pom.xml

<dependency>
	<groupId>com.fasterxml.jackson.core</groupId>
	<artifactId>jackson-core</artifactId>
	<version>2.9.0</version>
</dependency>
<dependency>
	<groupId>com.fasterxml.jackson.core</groupId>
	<artifactId>jackson-databind</artifactId>
	<version>2.9.0</version>
</dependency>
<dependency>
	<groupId>com.fasterxml.jackson.core</groupId>
	<artifactId>jackson-annotations</artifactId>
	<version>2.9.0</version>
</dependency>

​ 2.编写Controller逻辑，通过ObjectMappe中方法writeValueAsString(要转成json格式字符串的对象)，将对象转换为Json格式

//UserController.java

@Controller
public class UserController{
	@RequestMapping("/quick")
	@ResponseBody
	public String quick(){
		//User是一个实体类
		User user = new User();
		user.setName("张三");
		user.setAge(24);
		//Jackson库的转换工具类
		ObjectMapper objectMapper = new ObjectMapper();
		String json = objectMapper.writeValueAsString(user);
		return json;
	}
}

(5)回写数据-直接返回对象/集合-SpringMVC封装好的功能会将对象或集合自动转为Json格式

​ 要使用SpringMVC自带的该功能需要在spring-mvc.xml（配置组件）文件中进行配置，对处理器适配器进行功能增强

​ 注意：SpringMVC的组件在 Spring 容器启动时会自动加载并生效，不需要显式地注入

样例：配置处理器适配器，使其可以将返回对象自动转化为Json格式

​ 1.配置处理器适配器

//spring-mvc.xml

//配置处理器适配器：使得对象会自动转为Json格式
<bean id="jsonMessageConverter" class="org.springframework.web.servlet.mvc.method.annotation.RequestMappingHandlerAdapter">
	<property name="massageConverters">
		<list>
			<bean class="org.springframework.http.converter.json.MappingJackson2HttpMessageConverter"/>
		</list>
	</property>
</bean>

//以上处理器适配器的配置可以使用SpringMVC提供的注解驱动（也叫注解增强代替：增强@ResponseBody）
<mvc:annotation-driven/>

​ 2.在Controller中直接返回对象

//UserController.java

@Controller
public class UserController{
	@RequestMapping("/quick")
	@ResponseBody
	public String quick(){
		//User是一个实体类
		User user = new User();
		user.setName("张三");
		user.setAge(24);
		return user;
	}
}

5. SpringMVC请求参数的获取

​ 客户端发送请求时一般也会携带对应的参数，我们需要对请求参数进行获取，以便进一步对请求进行处理

​ 如果请求的数据是以 表单参数 或 查询参数 方式提交，Spring MVC 可以自动绑定到对象，而不需要 @RequestBody

(1)从请求中获得普通类型参数

​ 对于SpringMVC而言，Controller中的业务方法的参数名称与请求的参数name值一致，参数会自动映射匹配（数据类型也会自动进行转换）

http://localhost:8080/user/quick?username=zhangsan&age=18中的name值分别为username、age

​

样例：从请求中直接获得普通类型参数

​ 请求为：http://localhost:8080/user/quick?username=zhangsan&age=18

//UserController.java

@Controller
public class UserController{
	@RequestMapping("/quick")
	@ResponseBody
	public void quick(String username , int age){
		System.out.println(username);  //张三
		System.out.println(age);  //18
	}
}

(2)从请求中将参数打包为对象类型获得

​ 对于SpringMVC而言，Controller中的业务方法的对象参数里的属性与请求参数的name一致，参数值会自动注入给对象的属性（机制：数据绑定）

http://localhost:8080/user/quick?username=zhangsan&age=18中的name值分别为username、age

样例：从请求中直接把值注入给对象里的属性
//UserController.java

@Controller
public class UserController{
	@RequestMapping("/quick")
	@ResponseBody
	public void quick(User user){
		System.out.println(user.username);  //张三
		System.out.println(user.age);  //18
	}
}

//User类略

(3)SpringMVC注解-@RequestBody

​ @RequestBody：该注解写在形参前，用于处理 POST、PUT 等方法提交的 JSON 或 XML 数据，并将请求参数封装转换为 Java 对象

样例：获得ajax发送的请求，相应参数以List集合接收
//UserController.java

@Controller
public class UserController{
	@RequestMapping("/quick")
	@ResponseBody
	public void quick(@RequestBody List<User> userlist){
		System.out.println(userlist);  
	}
}

//仅需要User的类，List的类如果发的JSON是List数组就不需要，若发的JSON是包含List的对象就需要
//User类略

6. Spring MVC 静态资源访问配置与数据乱码问题解决

(1)SpringMVC对静态资源的处理方式

​ 在SpringBoot是自动配置好的，但是SpringMVC框架需要手动配置，如果不进行静态资源设置会使得访问静态资源（css、js等）不成功，因为在web.xml的文件中的前端控制器的配置是/，其是缺省的servlet，客户端发送请求找对应servlet进行处理，如果没有则全部交给缺省的进行处理，如下图所示：

配置静态资源处理器（spring-mvc.xml）
//spring-mvc.xml

//可以设置多组,mapping是请求访问的地址，location是实际资源存在根目录的位置
<mvc:resources mapping="/js/**" location="/js/">
//<mvc:resources mapping="/img/**" location="/img/">

(2)SpringMVC对请求中中文字符的处理配置

​ Tomcat8.5的服务器，对于GET请求是不会乱码的，但POST请求对于中文会出现乱码问题，通过配置全局过滤器可以解决乱码的问题

​ CharacterEncodingFilter是一个 Spring 提供的过滤器，它会在请求进入 Spring MVC 之前和响应返回客户端之前，确保编码的一致性。这个过滤器处理所有传入的请求和响应，强制它们使用指定的字符编码

配置全局过滤器（web.xml）
//web.xml

<filter>
	<filter-name>CharacterEncodingFilter</filter-name>
	<filter-class>org.springframework.web.filter.CharacterEncodingFilter</filter-class>
	<init-param>
		<param-name>encoding</param-name>
		<param-value>UTF-8</param-value>
	</init-param>
</filter>
//filter-mapping是配置请求映射指定什么请求通过，/*表示所有请求
<filter-mapping>
	<filter-name>CharacterEncodingFilter</filter-name>
	<url-pattern>/*</url-pattern>
</filter-mapping>

7. SpringMVC的其他注解

(1)@RequestParam-请求参数与形参绑定

​ @RequestParam：该注解写在形参前，用于当请求参数的名称和Controller的方法形参名称不一致时，通过@RequestParam(“请求参数名称一致的名称”)，让请求参数与方法形参进行绑定

样例：@RequestParam的使用

​ 请求为：http://localhost:8080/user/quick?username=zhangsan

//UserController.java

@Controller
public class UserController{
	@RequestMapping("/quick")
	@ResponseBody
	//当前参数只有一个且为value则可以省略value，写为@RequestParam("username")
	public void quick(@RequestParam(value="username") String name){
		System.out.println(username);  //张三
	}
}
@RequestParam的参数
  • value：设置映射，指定请求参数，将其映射到方法形参
  • required：指定请求时请求参数是否必须有，默认true，若请求中不包含此参数则报错
  • defaultValue：当请求中没有指定的请求参数时，则映射给方法形参指定的默认值

(2)@PathVariable-占位符中变量与形参绑定

​ @PathVariable：该注解写在形参前，用占位符{变量名}来获得请求中的常量，通过@PathVariable(“占位符的变量名”)，将占位符里的变量与方法形参进行绑定

样例：@PathVariable的使用

​ 请求为：http://localhost:8080/user/quick/98，这种请求是Restful风格的，你看没有这个?直接用/的

//UserController.java

@Controller
public class UserController{
	//用占位符的变量来与请求中常量绑定
	@RequestMapping("/quick/{score}")
	@ResponseBody
	//用@PathVariable让占位符的变量与方法形参进行绑定
	public void quick(@PathVariable("score") String myScore){
		System.out.println(myScore);  //98
	}
}

(3)@RequestHeader-获取请求头

​ @RequestHeader：写在方法形参前，通过@RequestHeader(“User-Agent”)获取请求头中的 User-Agent 值，并将该值与方法形参进行绑定，若请求头中没有该值，会抛异常

样例：@RequestHeader的使用
//UserController.java

@Controller
public class UserController{
	@RequestMapping("/quick")
	@ResponseBody
	//用@RequestHeader来获得请求头信息
	public void quick(@RequestHeader("User-Agent") String user_agent){
		System.out.println(user_agent);
	}
}

8. 自定义类型转换器

​ SpringMVC提供有一些常用的类型转换器，如接收到请求时，请求参数都是String类型，SpringMVC可以根据方法形参的类型进行自动转型（常用类型），但有些类型SpringMVC没有提供，那么就需要自定义类型转换器，如日期类型的数据转换

自定义类型转换器的开发步骤

​ 1.定义转换器类实现Converter接口（这是SpringMVC提供的转换器接口）

​ 2.在Spring-mvc.xml配置文件声明并识别转换器

样例：自定义时间类型转换器

​ 1.新建DateConverter.java，实现converter接口

//DateConverter.java

//Converter<S,T>接口，S代表要转换前的类型，后面的T代表要转成的类型
public class DataConverter implements Converter<String,Date>{
	//这是该接口内部要实现的方法，将String转为Date
	public Date convert(String dateStr){
		SimpleDateFormat format = new SimpleDateFormat("yyyy-MM-dd");
		Date date = null;
		try{
			date = format.parse(dateStr);
		}catch(Exception e){
			e.printStackTrace();
		}
		return date;
	}
}

​ 2.在Spring-mvc.xml配置文件中声明并识别自定义转换器，识别为替代原先SpringMVC提供的默认转换器

//Spring-mvc.xml

//声明转换器
<bean id="conversionService" class="org.springframework.context.support.ConversionServiceFactoryBean">
	<property name="converters">
		<list>
			//将DateConverter.java注入到转换器工厂
			<bean class="com.itheima.converter.DateConverter"></bean>
		</list>
	</property>
</bean>

//识别转化器（与1的注解功能启用独立）：将转化器工厂bean替换默认转换器
<mvc:annotation-driven conversion-service="conversionService"/>
