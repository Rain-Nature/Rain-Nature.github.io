Spring02-连接池与Spring注解

SSM
Publish Date:   2023-04-25

1. Spring配置连接池(数据源)

​ 连接池是对数据库连接的管理工具，常见的连接池有：DBCP、C3P0、BoneCP、Druid等

(1)连接池和数据库的区别

连接池
  • 连接池是一种用于管理数据库连接的机制，它通过提前创建一定数量的数据库连接，并在应用程序中重复利用这些连接，从而减少了频繁创建和销毁数据库连接的开销
  • 当应用程序需要与数据库进行交互时，连接池会从池中获取一个可用的连接，当操作完成后，该连接会被归还给连接池，以便其他请求使用
  • 连接池的主要作用是提高数据库操作的效率，特别是在高并发环境下。
数据库
  • 数据库是一个系统，用来存储和管理数据。它允许用户通过SQL语句来进行数据的增、删、改、查等操作
  • 常见的数据库有关系型数据库（如MySQL、PostgreSQL）和非关系型数据库（如MongoDB、Redis）

(2)连接池的手动连接使用(非Spring)

手动连接步骤

​ 1.添加连接池的依赖和数据库驱动的依赖（pom.xml）
​ 2.配置连接池基本连接信息和数据库连接信息（jdbc.properties）
​ 3.创建和初始化连接池（DataSourse.java）
​ 4.获取连接并使用（DataSourse.java）
​ 5.关闭连接（DataSourse.java）

手动连接实操

​ 1.在pom.xml文件中导入坐标，mysql、c3p0

//pom.xml

<dependencies>
	<dependency>
		<groupId>mysql</groupId>
		<artifactId>mysql-connector-java</artifactId>
		<version>5.1.32</version>
	</dependency>
	<dependency>
		<groupId>c3p0</groupId>
		<artifactId>c3p0</artifactId>
		<version>0.9.1.2</version>
	</dependency>
</dependencies>

​ 2.在resource文件夹下创建jdbc.properties文件，配置数据库连接信息

//jdbc.properties

jdbc.driver=com.mysql.jdbc.Driver
jdbc.url=jdbc:mysql://localhost:3306/test
jdbc.username=root
jdbc.password=root

​ 3.加载.properties文件来连接数据库，通过Java提供的ResourceBundle类以及该类提供的getBundle(“.properties文件的类加载路径且不需要写扩展名”)方法，来导入.properties文件。通过ResourceBundle对象的getString(“.properties文件中定义的key值”)，来获得对应的字符串

​ 4-5.获取连接/断开连接

//DataSourse.java

public void useDataSourse() throws Exception{
	//读取jdbc.properties配置文件
	ResourceBundle rb = ResourceBundle.getBundle("jdbc");
	String driver = rb.getString("jdbc.driver");
	String url = rb.getString("jdbc.url");
	String username = rb.getString("jdbc.username");
	String password = rb.getString("jdbc.password");
	
	//创建连接池对象，将连接参数配置
	ComboPooledDataSource dataSource = new ComboPooledDataSource();
	dataSource.setDriverClass(driver);
	dataSource.setJdbcUrl(url);
	dataSource.setUser(username);
	dataSource.setPassword(password);
	
	//获取连接
	Connection connection = dataSource.getConnection();
	System.out.println(connection);
	
	//关闭连接
	connection.close();
}

(3)连接池使用Spring容器连接

​ 对于c3p0连接池的对象dataSource的创建使用Spring容器完成

Spring容器连接步骤

​ 1.导入Spring坐标、连接池坐标、数据库坐标（pom.xml）
​ 2.创建数据库连接的配置文件（jdbc.properties）
​ 3.创建Spring的配置文件（applicationContext.xml）
​ 4.使用Spring容器获得连接池对象（SpringDataSourse.java）
​ 5.获取连接并使用（SpringDataSourse.java）
​ 6.关闭连接(SpringDataSourse.java)

Spring连接实操

​ 1.在pom.xml文件中导入坐标，mysql、c3p0、Spring

//pom.xml

<dependencies>
	<dependency>
		<groupId>mysql</groupId>
		<artifactId>mysql-connector-java</artifactId>
		<version>5.1.32</version>
	</dependency>
	<dependency>
		<groupId>c3p0</groupId>
		<artifactId>c3p0</artifactId>
		<version>0.9.1.2</version>
	</dependency>
	<dependency>
		<groupId>org.springframework</groupId>
		<artifactId>spring-context</artifactId>
		<version>5.0.5.RELEASE</version>
	</dependency>
</dependencies>

​ 2.在resource文件夹下创建jdbc.properties文件，配置数据库连接信息

//jdbc.properties

jdbc.driver=com.mysql.jdbc.Driver
jdbc.url=jdbc:mysql://localhost:3306/test
jdbc.username=root
jdbc.password=root

​ 3.在resource文件夹下创建Spring的applicationContext.xml配置文件，抽取properties文件，需要在命名空间添加引入Context，配置连接池的bean

<beans xmlns:context="http://www.springframework.org/schema/context" //命名空间添加这两句话用于引入Context
	   xsi:schemaLocation="http://www.springframework.org/schema/context http://www.springframework.org/schema/context/spring-context.xsd"> 
	
	//加载外部的properties文件到Spring容器中，当前文件在类加载路径下
	<context:property-placeholder location="classpath:jdbc.properties"/>
	
	<bean id="dataSource" class="com.mchange.v2.c3p0.ComboPooledDataSource">
		//name的值和set方法有关
		<property name="driverClass" value="${jdbc.driver}"></property>
		<property name="jdbcUrl" value="${jdbc.url}"></property>
		<property name="user" value="${jdbc.username}"></property>
		<property name="password" value="${jdbc.password}"></property>
	</bean>
</beans>

​ 4.使用Spring容器获得连接池对象

​ 5-6.获取连接/断开连接

//SpringDataSourse.java

public void useSpringDataSourse throws Exception{
	//获得连接池对象
	ApplicationContext app = new ClassPathXmlApplicationContext("applicationContext.xml");
	DataSource dataSource = (DataSource) app.getBean("dataSouce");
	
	//获取连接
	Connection connection = dataSource.getConnection();
	System.out.println(connection);
	
	//关闭连接
	connection.close();
}

2. Spring注解开发

​ Spring是轻代码重配置的框架，配置比较繁重影响开发效率，注解完全代替xml配置文件可以简化配置，提高开发效率

(1)Spring注解开发的准备工作

​ 要使用注解需要提前告诉Spring，通过在applicationContext.xml文件中配置组件扫描，指定哪个包及其子包下的Java文件需要进行扫描以便识别使用注解配置的类、字段和方法

语法：配置组件扫描

​ base-package="包名"，其会扫描该包下所有的子包，包下所有使用了注解的都会被扫描到

//applicationContext.xml，是在下方，不是命名空间配置，这个也需要引入context命名空间

<context:component-scan base-package="com.itheima"></context:component-scan>

(2)Spring原始注解(核心注解)的介绍

​ Spring原始注解主要用于代替Spring的xml文件来配置和管理 Spring 容器中的 Bean，处理依赖注入、事务管理、事件处理等任务

常用的原始注解
注解 说明
@Component 写在类上，标记这个类作为 Spring 容器中的 Bean
@Controller 写在作为Spring MVC 控制器的类上，与@Component效果一样，仅作为语义区分
@Service 写在作为服务层的类上，与@Component效果一样，仅作为语义区分
@Repository 写在作为数据访问层的类上，与@Component效果一样，仅作为语义区分
@Autowired 写在字段、构造方法或set方法上，自动将一个Spring容器的Bean 注入到另一个 Bean
@Qualifier @Autowired根据类型注入Bean，当有多个相同类型的Bean时，在其下方使用该注解指定要注入的 Bean 的名字
@Value 写在字段上，用于注入普通值或外部配置值（如 application.properties或 application.yml文件中的属性）
@Scope 写在类上，定义 Bean 的作用域
@PostConstruct 写在方法上，该方法在该类的Bean初始化后自动执行
@PreDestroy 写在方法上，该方法在容器销毁该类的Bean前自动执行

(3)Spring的原始注解详解

@Component(“唯一标识”)

​ 使用注解@Component("唯一标识")等同在xml文件中对bean的配置，在需要Spring容器管理的类上方写注解，

​ 注意：唯一标识可以不写，如@Component

样例：@Component的使用

//UserDaoImpl.java

//等价于xml文件：<bean id="userDao" class="com.itheima.dao.impl.UserDaoImpl"></bean>
@Component("userDao")
public class UserDaoImpl implements UserDao{
	public void save(){
		System.out.println("save running...");
	}
}
@Autowired

​ 使用注解@Autowired等同于在xml文件中的propety属性的注入配置，写在字段上，Spring 会自动为该字段注入一个 Bean（前提是容器中有与字段类型匹配的 Bean），没有找到合适的或找到多个其会抛出异常

​ 一个bean可省：如果存在多个符合条件的 bean可以在其下方加@Qualifier("唯一标识")指定注入bean的名字

样例：@Autowired的使用

//UserServiceImpl.java

//等价于xml文件：<bean id="userService" class="com.itheima.service.impl.UserServiceImpl">
@Component("userService")
public class UserServiceImpl implements UserService{
	//等价于xml文件：<property name="userDao" ref="userDao"></property>
	@Autowired
	@Qualifier("userDao")
	private UserDao userDao;
	
	//注入字段不需要额外写set方法
}
@Controller，@Service，@Repository

​ 他们都属于特殊类型的@Component注解，功能与其一致，作用是增强代码的可读性：@Controller—web层；@Service—service层；@Repository—dao层

​ 注意：他们三个都可以写唯一标识，如@Controller(“userController”)

样例：@Controller，@Service，@Repository的使用

//UserController.java

@Controller
public class UserController{
    // 控制器逻辑
}
//UserServiceImpl.java

@Service
public class UserServiceImpl implements UserService{
    // 业务逻辑
}
//UserDaoImpl.java

@Repository
public class UserDaoImpl implements UserDao{
    // 数据访问逻辑
}
@Value(“要注入的值”)

​ @Value用于注入属性值，可以值直接注入bean或者将外部的配置值（如application.properties或application.yml文件中的属性）注入到 Bean 的字段中，在值注入中虽然值是你提供的，但还是Spring来进行注入，而使用外部配置值需要将这些文件提前注入到Spring容器中

样例：@Value(“要注入的值”)的使用

//UserDaoImpl.java

@Component("userDao")
public class UserDaoImpl implements UserDao{
	//将String类型的itcast注入it，直接注入
	@Value("itcast")
	private String it;
	
	//Spring容器中要提前加载jdbc.properties文件才可以通过`${}`对文件里的内容进行注入
	@Value("${jdbc.drive}")
	private String driver;
	
}
@Scope(“属性值”)

​ @Scope定义bean的范围，其写在类的上方，用来设置bean是单例还是多例

样例：@Scope(“属性值”)的使用

//UserServiceImpl.java

@Component("userService")
//设置创建的bean是多例，单例设置为@Scope("singleton")
@Scope("prototype")
public class UserServiceImpl implements UserService{
	//...
}
@PostConstruct和@PreDestroy

​ 写在类中的方法前，用于设置bean在创建或者销毁时要执行的方法

//UserServiceImpl.java

@Component("userService")
public class UserServiceImpl implements UserService{

	//配置该方法在bean创建时调用
	@PostConstruct
	public void init(){
		System.out.println("Service类的bean初始化调用");
	}
	
	//配置该方法在bean销毁前调用
	@PreDestroy
	public void destroy(){
		System.out.println("Service类的bean销毁前调用");
	}
}

(4)Spring3新注解(配置注解)介绍

​ 原始注解不能完全代替xml配置文件，通过使用@Configuration注解的类可以充当Spring的Java配置文件，取代传统的 applicationContext.xml配置文件

使用Spring新注解替代的配置如下：
  • 非自定义的类的bean导入：如需要将第三方jar包中的类配置到Spring容器中
  • 加载properties文件的配置：< context:property-placeholder >
  • 组件扫描的配置：< context:component-scan >
  • 引入其他文件：< import >
Spring3的新注解
注解 说明
@Configuration 写在类上，用于标注该类为Java的配置类用于代替Spring的xml配置文件
@ComponentScan 写在Java的配置类上，用于指定自动扫描的包及子包
@Bean 写在方法上，将该方法的返回值交由Spring容器管理
@PropertySource 写在类上，用于加载外部的属性文件（如 .properties文件）
@Import 写在Java的配置类上，用于将其他的配置类导入到该配置类

(5)Spring新注解详解

@Configuration

​ @Configuration用于标识一个类作为配置类，Spring 容器就会自动扫描这个类，并将它作为配置类来处理。配置类是用来定义 Spring 容器中的bean 配置的，这些类通常包含了多个@Bean注解的方法，这些方法的返回值会交由Spring 容器管理

样例：@Configuration的使用

//SpringConfiguration

//标志该类是Spring的核心配置类
@Configuration
public class SpringConfiguration{
	@Bean
	//方法...
}
面试题：@Configuration和@Component中@Bean的区别

​ @Configuration中的@Bean方法的返回值直接交给Spring管理（必定单例），而@Component中的Bean是作为普通方法调用，可能造成多例问题

//情况1：@Configuration，推荐
@Configuration
public class MyConfig {

    @Bean
    public B b() {
        System.out.println("new B()");
        return new B();
    }

    @Bean
    public A a() {
        return new A(b()); // ✅ 实际用的是 Spring 容器中的 B 实例（只 new 一次）
    }
}

//情况2：@Component，不推荐
@Component
public class MyComponent {

    @Bean
    public B b() {
        System.out.println("new B()");
        return new B();
    }

    @Bean
    public A a() {
        return new A(b()); // ❌ 实际调用方法，会再 new 一个 B()
    }
}
@ComponentScan(“包名”)

​ @ComponentScan用于指定自动扫描的包及其子包，其相当于xml文件中的组件扫描

样例：@ComponentScan(“包名”)的使用

//SpringConfiguration

@Configuration
//等价于<context:component-scan base-package="com.itheima"/>
@ComponentScan("com.itheima")
public class SpringConfiguration{
	//...
}
@Bean

​ @Bean用于将当前方法的返回值以当前方法名为id存储到Spring容器中，可以使用@Bean(“id”)来显示的指定存储的bean的唯一id

样例：@Bean的使用

//SpringConfiguration

@Configuration
@ComponentScan("com.itheima")
public class SpringConfiguration{
	
	//将当前方法的返回值以id为getDataSource存储在Spring容器中
	@Bean
	public DataSource getDataSource() throws PropertyVetoException{
		//这里与xml文件不同的是，xml是将ComboPooledDataSource的dataSource放入Spring容器，这里是执行后的返回值
		ComboPooledDataSource dataSource = new ComboPooledDataSource();
        dataSource.setDriverClass("com.mysql.jdbc.Driver");
        dataSource.setJdbcUrl("jdbc:mysql://localhost:3306/test");
        dataSource.setUser("root");
        dataSource.setPassword("root");
        return dataSource;
	}
}
@PropertySource(“文件路径”)

​ @PropertySource用于加载外部的属性文件（例如 .properties文件）到Spring容器中

样例：@PropertySource(“文件路径”)的使用

//SpringConfiguration

@Configuration
@ComponentScan("com.itheima")
//等价与：<context:property-placeholder location="classpath:jdbc.properties"/>
@PropertySource("classpath:jdbc.properties")
public class SpringConfiguration{
	//使用@value即可读取.properties文件里的信息
	@Value("${jdbc.driver}")
    private String driver;
    
    @Value("${jdbc.url}")
    private String url;
    
    @Value("${jdbc.username}")
    private String username;
    
    @Value("${jdbc.password}")
    private String password;
}
@Import(“配置类的名字.class”)

​ @Import用于将一个或多个外部的配置类或者资源（如 Bean 定义：bean定义会随着导入外部的配置类一起导入）导入到当前配置类中

样例：@Import(“配置类的名字.class”)的使用

//SpringConfiguration

@Configuration
@ComponentScan("com.itheima")
//等价与：<import resource="applicationContext-dataSource.xml"/>
@Import(DataSourceConfiguration.class)
//多个导入：@Import({xxx.class,yyy.class,zzz.class})
public class SpringConfiguration{
	//...
}

(6)配置类来加载Spring容器对象

​ 在使用applicationContext.xml文件时，是通过加载xml文件来获得Spring容器对象，而使用配置类后同样需要加载配置类

样例：配置类的加载

//main.java

public static void main(String[] args){
	//获得Spring容器对象，之前用xml是：ApplicationContext app = new ClassPathXmlApplicationContext(applicationContext.xml);
	ApplicationContext app = new AnnotationConfigApplicationContext(SpringConfiguration.class);
	UserService u = app.getBean(UserService.class);
	u.save();
}
