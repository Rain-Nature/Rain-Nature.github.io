(2)RESTful风格介绍

​ REST全称（Resource Representational State Transfer）主张万物皆资源，对Web的操作就是对资源的操作，在url中使用资源名作为路径，用http动词来表示对资源的操作

  • Resource：资源
  • Representational：表现形式，比如JSON、XML、JPEG等
  • State Transfer：状态变化，通过http动词（GET、POST、PUT、DELETE）实现

//EmployeeService.java

public interface EmployeeService {

	//根据id查询用户
	Employee getEmployeeById(Long id);

	//保存用户
	void saveEmployee(Employee employee);

	//删除用户
	void deleteEmployee(Long id);

	//更新用户
	void updateEmployee(Employee employee);
	
	//查询所有用户
	List<Employee> getList();
}

//EmployeeServiceImpl.java
@Service
public class EmployeeServiceImpl implements EmployeeService {

	@Autowired
	EmployeeDao employeeDao;

	@Override
	public Employee getEmployeeById(Long id) {
		Employee employee = employeeDao.getEmployeeById(id);
		return employee;
	}

	@Override
	public void saveEmployee(Employee employee) {
		employeeDao.addEmployee(employee);
	}

	@Override
	public void deleteEmployee(Long id) {
		employeeDao.deleteEmployee(id);
	}

	@Override
	public void updateEmployee(Employee employee) {
		//防null处理
		Long id = employee.getId();
		if (id == null) {
			return;
		}
		//1、去数据库查询employee原来的值
		Employee employeeold = employeeDao.getEmployeeById(id);

		//2、把页面带来的覆盖原来的值，页面没带来的值保持原状
		//姓名
		if (StringUtils.hasText(employee.getName())) {  //判断name有值（不是null，不是空串，不是空白字符）
			//把数据库的值改为页面传来的值
			employeeold.setName(employee.getName());
		}

		//邮箱
		if (StringUtils.hasText(employee.getEmail())) {
			//把数据库的值改为页面传来的值
			employeeold.setEmail(employee.getEmail());
		}

		//性别
		if (StringUtils.hasText(employee.getGender())) {
			//把数据库的值改为页面传来的值
			employeeold.setGender(employee.getGender());
		}

		//地址
		if (StringUtils.hasText(employee.getAddress())) {
			//把数据库的值改为页面传来的值
			employeeold.setAddress(employee.getAddress());
		}

		//年龄
		if (employee.getAge() != null) {
			employeeold.setAge(employee.getAge());
		}

		//工资
		if (employee.getSalary() != null) {
			employeeold.setSalary(employee.getSalary());
		}

		//3、对数据库进行修改
		employeeDao.updateEmployee(employeeold);
	}
	
	@Override
	public List<Employee> getList() {
		return employeeDao.getList();
	}
}

(4)REST风格的CRUD案例的优化-统一返回R对象

​ Controller层方法返回值不统一，有String类型，也有Employee类型，对于前端而言希望得到的是一个统一返回

统一返回内容格式（R对象格式）
//code：业务的状态码，200是成功，其余都是失败，前后端会一起商定不同的业务状态码在前端需要显示的效果
//msg：服务端返回给前端的提示消息
//data：服务器返回给前端的数据，一个数据放对象，多个数据放数组

//返回内容格式
{
	"code":300,
	"msg":"余额不足",
	"data":null
}
前端统一处理（R对象）

​ 1.前端发送请求，接受服务器数据

​ 2.判断状态码，成功显示数据，失败显示提示信息或执行其他操作

使用R对象的开发步骤

​ 1.新建common包，在包下新建类R

​ 2.在Controller的方法中将返回的数据封装为R返回

样例：使用统一返回对象（R）对Controller层优化

​ 1.新建common包，新建类R.java

//R.java

@Data
public class R<T> {
	private Integer code;
	private String msg;
	private T data;

	//为了不再Controller再组装R，所以R需要提供一个返回成功的静态方法
	public static <T> R<T> ok(T data) {
		R<T> r = new R<T>();
		r.setCode(200);
		r.setMsg("ok");
		r.setData(data);
		return r;
	}

	//重载一个不返回任何数据的方法
	public static R ok() {
		R r = new R();
		r.setCode(200);
		r.setMsg("ok");
		return r;
	}
	
	//有失败逻辑时调用的方法
	public static <T> R<T> error(Integer code ,String msg , T data) {
		R<T> r = new R<T>();
		r.setCode(code);
		r.setMsg(msg);
		r.setData(data);
		return r;
	}
}

2. 跨域

​ 跨域（CORS, Cross-Origin Resource Sharing） 是指 浏览器 出于 同源策略（Same-Origin Policy）的安全限制，阻止从一个域（A）向另一个域（B）发起请求

跨域问题

​ 协议、域名、端口不完全相同，就会触发跨域问题

请求地址（前端） 后端 API 地址 是否跨域？ 原因
http://localhost:3000 http://localhost:8080 ✅ 跨域 端口不同
http://example.com https://example.com ✅ 跨域 协议不同
http://sub.example.com http://example.com ✅ 跨域 子域名不同
http://example.com:80 http://example.com:8080 ✅ 跨域 端口不同
http://example.com http://example.com ❌ 同源 完全相同
后端解决跨域问题-允许前端的请求跨域

​ 原理：在服务器给浏览器的响应头中添加字段，Access-Control-Allow-Origin = *，SpringBoot通过注释实现

​ 在SpringBoot中只需要在类名或方法上添加注释@CrossOrigin，该类或方法就可以跨域访问

​ 前端对复杂的跨域请求会发送两次

​ 1.options请求：预检请求，浏览器会先发送options请求，询问服务器是否允许当前域名进行跨域访问

​ 2.真正的请求：POST、DELETE、PUT等

3. Spring开发中的常用注解

(1)@Data

​ @Data 写在类前，它是 Lombok 提供的一个注解，用于自动生成常见的 Java 类方法，从而减少样板代码。该注解可以自动为你的类生成 getter、setter、toString()、equals() 和 hashCode()等方法

样例：@Data的使用

​ @SpringBootTest写在类前，它用于测试 Spring Boot 应用的集成测试（不是单元测试）。这个注解会启动整个 Spring 上下文，它支持依赖注入，通常用于 Web、服务层或 数据库层等需要集成测试的场景

样例：@SpringBootTest的使用
//MyApplicationTests.java

@SpringBootTest
public class MyApplicationTests {

    @Test

    }

​ @Test写在方法前，它是 JUnit 框架的核心注解，用于标记测试方法。在 Spring Boot 中，常与@SpringBootTest 等注解配合使用，用于集成测试，有该注解标记的方法可以直接运行

(4)@RestController

​ @RestController写在类前，它等价于@Controller+@ResponseBody

5. SpringMVC拦截器

​ Spring MVC 拦截器（HandlerInterceptor）用于在 请求到达 Controller 之前、执行 Controller 方法之后、视图渲染之前 进行拦截和处理

​ 拦截器也是AOP思想的具体实现

(1)SpringMVC对请求的处理流程

请求 → 监听器（Listener） → 过滤器（Filter） → 拦截器（Interceptor） → Controller 处理请求

  • 监听器（Listener）：监听应用/Session/请求的生命周期，全局事件处理
  • 过滤器（Filter）：拦截 HTTP 请求/响应，处理权限、日志、编码、跨域
  • 拦截器（Interceptor）：拦截 Spring MVC 方法（Spring业务逻辑），适用于权限校验、日志、AOP 逻辑

(2)监听器、拦截器和过滤器的使用场景

需求 监听器（Listener） 过滤器（Filter） 拦截器（Interceptor）
统计在线人数 ✅ 适用 ❌ 不适用 ❌ 不适用
初始化系统资源 ✅ 适用 ❌ 不适用 ❌ 不适用
跨域（CORS）处理 ❌ 不适用 ✅ 适用 ❌ 不适用
登录认证 ❌ 不适用 ✅ 适用 ✅ 适用
权限控制 ❌ 不适用 ✅ 适用 ✅ 适用（推荐）
请求日志记录 ❌ 不适用 ✅ 适用 ✅ 适用
修改请求参数 ❌ 不适用 ✅ 适用 ✅ 适用

样例：SpringMVC拦截器的使用

​ 1.新建MyInterceptor.java实现HandlerInterceptor接口，该接口中的方法都是使用default修饰符来空实现，所以需要将要使用的方法覆盖，一共有三个方法：preHandle()**：请求进入Controller之前执行（适用于权限校验、参数校验），若返回false后面的所有流程都不会执行；postHandle()：Controller方法执行后执行（适用于修改返回数据/视图）；afterCompletion()**： 视图渲染完成后执行（适用于日志记录、异常处理）

//MyInterceptor.java

public class MyInterceptor implements HandlerInterceptor{
	//在Controller层方法执行之前执行
	public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler){
		// 检查 Session 中是否有用户
        String user = (String) request.getSession().getAttribute("user");
        if (user == null) {
            response.sendRedirect("/login"); // 未登录，重定向到登录页面
            return false; // 拦截请求：不会再执行Controller的方法
        }
        return true; // 继续请求
	}
	
	//在Controller层方法执行之后，视图对象返回之前执行
	public void postHandle(HttpServletRequest request, HttpServletResponse response, Object handler, ModeAndView modeAndView){
		System.out.println("【拦截器】postHandle 执行...");
	}
	
	//在视图渲染完成之后执行
	public void afterCompletion(HttpServletRequest request, HttpServletResponse response,  Object handler, Exception ex){
		System.out.println("【拦截器】afterCompletion 执行...");
	}
}

​ 2.在spring-mvc.xml文件里配置拦截器，<mvc:interceptors> 标签内部可以配置多个拦截器，<mvc:interceptor>表示一个拦截器，<mvc:mapping>中配置对哪些资源执行拦截操作

//spring-mvc.xml

//配置拦截器
<mvc:interceptors>
    <mvc:interceptor>
    	// /**表示对所有的方法都进行拦截操作
        <mvc:mapping path="/secure/**"/>
        //放行/login
        <mvc:exclude-mapping path="/login"/>
        <bean class="com.itheima.interceptor.MyInterceptor"/>
    </mvc:interceptor>
</mvc:interceptors>

6. 异常处理

(1)异常处理的方式

  • 编程式异常处理
    • try-catch、throw、exception
  • 声明式异常处理（开发推荐）
    • SpringMVC提供了@ExceptionHandle、@ControllerAdvice等便捷的声明式注解来进行异常处理

(2)SpringMVC中本类异常处理的注解-@ExceptionHandle(异常类的全限定名.class)

​ @ExceptionHandle写在Controller类中的方法上，如果这个Controller类中出现异常，其会自动在本类中找@ExceptionHandle标记的方法（可以是多个），有对应的异常处理则执行对应方法，如果多个能处理则精确的优先

样例：@ExceptionHandle的使用

​ 请求路径：http://localhost:8080/hello?i=0

//HelloController.java

@RestController
public class HelloController {

	@GetMapping("/hello")
	public R hello(Integer i){
		int b = 10/ i;
		return R.ok(b);
	}

	//对算术异常 (ArithmeticException)进行管理，只要该Controller中发生算数异常则调用这个异常处理方法
	@ExceptionHandler(ArithmeticException.class)
	public R handleArithmeticException(ArithmeticException ex){
		return R.error(100,"执行异常",ex.getMessage());
	}
}

(3)SpringMVC中全局异常处理的注解-@ControllerAdvice

​ @ControllerAdvice写在类上，表示这个类是一个全局“助手”类，专门用来辅助所有 @Controller处理一些通用逻辑。它是 Spring MVC 提供的全局控制增强机制（AOP思想），用于对所有 @Controller 进行统一处理，它的应用场景如下：

  • 全局异常处理（最常见用法）
  • 全局数据绑定（给所有 Controller 统一绑定数据）
  • 全局数据格式转换（参数解析、类型转换）
样例：使用@ControllerAdvice+@ExceptionHandle实现全局异常处理

​ 新建advice包作为增强包，在该包下新建GlobalExceptionHandle.java作为全局异常处理类

//GlobalExceptionHandle.java

@ControllerAdvice
public class GlobalExceptionHandle {
        
	@ResponseBody
	@ExceptionHandler(Exception.class)
	public R error(Exception e){
		return R.error(500,e.getMessage(),null);
	}
}

(4)异常处理优先级与保底机制

异常处理的优先级

本类 > 全局

精确 > 模糊

异常处理的默认行为

​ 如果出现了异常，本类和全局都不能处理，SpringBoot底层对SpringMVC有兜底处理机制，这是自适应的（如果是浏览器则返回错误页面，如果是Postman则返回以json为格式的错误数据）

(5)异常处理在实际开发（商用）中的使用

​ 前后端项目中，前端感知异常状态，后端只编写正确的业务逻辑，如果出现业务问题，后端通过抛业务异常（需要自定义）的方式提前中断业务逻辑

异常处理的使用步骤（商用）

​ 1.编写异常枚举类BizExceptionEnume，用于设定异常码以及对应的异常信息（这些是不变的）

​ 2.编写自定义异常类BizException，里面需要有异常码和异常信息字段，并通过构造器允许赋值

​ 3.在Sevice层中，对于非正常的业务流程new一个自定义异常对象

​ 4.在异常处理类中，获得自定义异常对象，并将状态码和信息返回给前端

样例：异常处理的实际使用

​ 1.编写枚举类BizExceptionEnume.java，设定异常码以及码对应的信息

//BizExceptionEnume.java

public enum BizExceptionEnume {

	//ORDER_xxx：订单模块相关异常
	ORDER_CLOSED(10001,"订单已关闭"),
	ORDER_NOT_EXIST(10002,"订单不存在"),
	ORDER_TIMEOUT(10003,"订单超时");

	@Getter
	private Integer code;
	@Getter
	private String message;

	private BizExceptionEnume(Integer code, String message) {
		this.code = code;
		this.message = message;
	}
}

​ 2.新建一个exception包，在里面自定义异常BizException.java

//BizException.java

@Data
public class BizException extends RuntimeException {

	private Integer code;
	private String message;

	public BizException(Integer code, String message) {
		super(message);
		this.code = code;
		this.message = message;
	}

	public BizException(BizExceptionEnume bizExceptionEnume) {
		super(bizExceptionEnume.getMessage());
		this.code = bizExceptionEnume.getCode();
		this.message = bizExceptionEnume.getMessage();
	}
}

​ 3.在service层中将错误流程new一个异常对象，并抛出异常，这里用的是 restfu开发的service作为样例，优化的是它的update方法，之前遇到错误是直接return

//EmployeeServiceImpl

public void updateEmployee(Employee employee) {
	//防null处理
	Long id = employee.getId();
	if (id == null) {
		//终端业务：这里原先是直接return
		throw new BizException(BizExceptionEnume.ORDER_CLOSED);
	}
	//1、去数据库查询employee原来的值
	Employee employeeold = employeeDao.getEmployeeById(id);

	//2、把页面带来的覆盖原来的值，页面没带来的值保持原状
	//姓名
	if (StringUtils.hasText(employee.getName())) {  //判断name有值（不是null，不是空串，不是空白字符）
		//把数据库的值改为页面传来的值
		employeeold.setName(employee.getName());
	}

	//邮箱
	if (StringUtils.hasText(employee.getEmail())) {
		//把数据库的值改为页面传来的值
		employeeold.setEmail(employee.getEmail());
	}

	//性别
	if (StringUtils.hasText(employee.getGender())) {
		//把数据库的值改为页面传来的值
		employeeold.setGender(employee.getGender());
	}

	//地址
	if (StringUtils.hasText(employee.getAddress())) {
		//把数据库的值改为页面传来的值
		employeeold.setAddress(employee.getAddress());
	}

	//年龄
	if (employee.getAge() != null) {
		employeeold.setAge(employee.getAge());
	}

	//工资
	if (employee.getSalary() != null) {
		employeeold.setSalary(employee.getSalary());
	}

	//3、对数据库进行修改
		employeeDao.updateEmployee(employeeold);

}
