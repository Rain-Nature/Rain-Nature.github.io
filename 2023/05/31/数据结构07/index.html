数据结构
Publish Date:   2023-05-31

树

1. 树的定义

​ 专业的定义：

​ 1. 有且只有一个称为根的节点
​ 2. 有若干个互不相交的子树，这些子树本身也是一颗树

​ 通俗的定义：

​ 1. 树是由节点和边(指针)组成
​ 2. 每个节点只有一个父节点但可以有多个子节点
​ 3. 但有一个节点例外，该节点没有父节点，此节点称为根节点

2. 树的专业术语

  • 节点：树的节点由数据元素及其若干分支组成
  • 父节点：节点上面紧挨着的节点
  • 子节点：父节点下面一层的节点
  • 兄弟：父节点下的所有子结点互为兄弟结点
  • 子孙：以某节点为根的子树中的任一节点
  • 深度：从根节点到最底层节点的层数，根节点是第一层
  • 叶子节点：没有子节点的节点
  • 非终端节点：非叶子节点
  • 度：子节点的个数

​

3. 树的分类

  • 一般树

    任意一个节点的子节点的个数都不受限制

  • 二叉树

    任意一个节点的子节点的个数最多两个，且子节点的位置不可更改

  • 森林

    n个互不相交的树的集合

4. 二叉树的分类

  • 一般二叉树

  • 满二叉树

    在不增加树的层数的情况下，无法再多添加一个节点的二叉树

  • 完全二叉树

    如果只是删除了满二叉树最底层最右边的连续若干个节点，这样形成的二叉树就是完全二叉树

5. 树的存储

(1) 二叉树的存储(比较成熟)

​ 连续存储[必须是完全二叉树]

​ 优点：查找某个节点的父节点和子节点(也包括判断有没有子节点)速度很快，时间复杂度O(1)

​ 缺点：耗用内存空间过大

​ 当二叉树不是完全的时候需要去补全，先转换为满二叉树再转化为完全二叉树，如下图所示，蓝色为补全节点，黄框可以不补全

​ 存蓝点的原因是方便还原树的形状

​ 通过先序、中序、后序可以将树从非线性结构转换为线性结构，使用数组存储时，不可以 只存储有效节点，因为只通过先序、中序、后序的任一一种无法还原原先的二叉树

​ 链式存储(不是链表存)

​ 每一个节点分成三个部分，左右存指针，中间存数据

​ 内存浪费：空指针浪费n+1个

​ 找父节点也很困难

(2) 一般树的存储方法

  1. 双亲表示法

    使用数组的方式进行存储，一个存储块分为两个区域，一个区域存节点名，另一个区域存父节点所在的数组元素的下标

    双亲表示法求父节点方便

  2. 孩子表示法

    使用数组的方式进行存储，也是分成两个区域，一个区域存节点，另一个区域是链表存指针依次指向其子节点

    孩子表示法求子节点方便

  3. 孩子双亲表示法

    把上面两种方法结合在一起，在数组中存放三个域

    求父节点和子节点都很方便

  4. 二叉树表示法

    因为在实现过程中对于孩子双亲表示法不方便，所以使用了二叉树表示法，二叉树方便使用代码来表示，详细见第6大节

(3) 森林的存储

​ 一般是把森林转化为二叉树再进行存储

​ 把森林中每一个树的根节点当作兄弟节点，然后像一般树转化为二叉树那样将森林转换为二叉树

6. 普通树的二叉树表示法

​ 把一个普通树转化成二叉树来存储

​ 转换方法：设法保证任意一个节点的左指针域指向它的第一个孩子，右指针域指向它的兄弟节点，只要能满足此条件，就可以把一个普通树转化为二叉树

​ 一个普通树转化成的二叉树一定没有右子树

7. 二叉树的操作

​ 1. 遍历，先中后的遍历都是指二叉树的遍历
​ 2. 已知两种遍历序列求原始二叉树

​ 通过先中或中后遍历序列可以还原二叉树，但先后不能还原出原始二叉树

​ 注意：遍历时当访问到根节点时将该节点记录

(1). 二叉树的先序遍历

​ 先序遍历[先访问根节点]

​ 递归的进行(先访问树的根节点，再访问左子树，再访问右子树)

(2). 二叉树的中序遍历

​ 中序遍历[中间访问根节点]

​ 递归的进行(先遍历树的左子树，再访问根节点，再遍历右子树)

(3). 二叉树的后序遍历

​ 后序遍历[最后访问根节点]

​ 递归的进行(先遍历树的左子树，再遍历右子树，再访问根节点)

(4). 已知先序和中序求后序

​ 首先使用先序和中序将二叉树求出来，然后再通过二叉树求出后序

​ 使用先序确定节点顺序(找每个子树的根节点)，中序确定是左子树还是右子树

​ 从树可以得出后序为：DECBHGFA

(5). 已知中序和后序求先序

​ 首先使用中序和后序将二叉树求出来，然后再通过二叉树求出先序

​ 使用后序确定节点顺序(找根节点，子树中最后出现的节点为根节点)，中序确定是左子树还是右子树，把大的树不断递归上面方法变成小树最后得出具体节点

​ 从树可以得出先序为：ABCDEFGH

8. 树的应用

  • 树是数据库中数据组织一种重要形式

  • 操作系统子、父进程的关系本身就是一棵树

  • 面向对象语言中类的继承关系

  • 哈夫曼树：一个事物有n种可能取值，每一种取值又有不同的概率，如何使得工作的效率达到最高

9. 链式二叉树遍历的具体程序演示

​ 静态链式二叉树的创建与先序、中序、后序遍历代码

#include <stdio.h>
#include <stdlib.h>
#include <malloc.h>

//二叉树英文叫Binart Tree
typedef struct BTNode{
    char data;
    //p是指针 L是左 child是子节点
    struct BTNode *pLchild;
    struct BTNode *pRchlid;
}BTN,*PBTN;

//该函数为静态的创造树，但内存是需要动态分配的
PBTN CreateBTree(void);
//先序遍历
void  PreTraverseBTree(PBTN pT);
//中序遍历
void  InTraverseBTree(PBTN pT);
//后序遍历
void  PostTraverseBTree(PBTN pT);


PBTN CreateBTree(void){
    PBTN pA = (PBTN)malloc(sizeof(BTN));
    PBTN pB = (PBTN)malloc(sizeof(BTN));
    PBTN pC = (PBTN)malloc(sizeof(BTN));
    PBTN pD = (PBTN)malloc(sizeof(BTN));
    PBTN pE = (PBTN)malloc(sizeof(BTN));

    pA->data = 'A';
    pA->pLchild = pB;
    pA->pRchlid = pC;
    pB->data = 'B';
    pB->pLchild = pB->pRchlid = NULL;
    pC->data = 'C';
    pC->pLchild = pD;
    pC->pRchlid = NULL;
    pD->data = 'D';
    pD->pLchild = NULL;
    pD->pRchlid = pE;
    pE->data = 'E';
    pE->pLchild = pE->pRchlid = NULL;

    //返回根节点的地址
    return pA;
}

void PreTraverseBTree(PBTN pT){
    if(NULL != pT){
        printf("%c\n",pT->data);


        if(NULL != pT->pLchild){
            PreTraverseBTree(pT->pLchild);
        }

        if(NULL != pT->pRchlid){
            PreTraverseBTree(pT->pRchlid);
        }
    }



}

void  InTraverseBTree(PBTN pT){
    if(NULL != pT){
        if(NULL != pT->pLchild){
            InTraverseBTree(pT->pLchild);
        }
        printf("%c\n",pT->data);
        if(NULL != pT->pRchlid){
            InTraverseBTree(pT->pRchlid);
        }
    }

}

void  PostTraverseBTree(PBTN pT){
    if(NULL != pT){
        if(NULL != pT->pLchild){
            PostTraverseBTree(pT->pLchild);
        }
        if(NULL != pT->pRchlid){
            PostTraverseBTree(pT->pRchlid);
        }
        printf("%c\n",pT->data);
    }
}

int main()
{
    PBTN pT = CreateBTree();
    PostTraverseBTree(pT);
    return 0;
}
