JS高级03-模块化

JS高级
Publish Date:   2025-01-12

1. 模块化

​ 将程序文件依据一定规则拆分为多个文件，每一个文件为一个模块，模块中的数据是私有的，模块之间相互隔离，但可以把模块内的指定数据交出去，供其他模块使用

(1)模块化会遇到的问题

  • 全局污染(命名冲突)问题：

    引入js 模块时，若有声明同名的方法，后面的会把前面的给覆盖掉

  • 依赖混乱问题：

    引入js模块时，有些要先执行，如果引入顺序错误会使得代码无法正常运行

  • 数据安全问题：

    数据没有进行封装，仅暴露接口函数，但实际上获得对象时，所有数据都已经获得

​ 使用模块化规范会解决上述的所有问题，全局污染问题详细阅读整个blog；依赖混乱问题，通过模块化使得模块之间呈树状结构所以不会混乱；数据安全问题，因为模块化规范是暴露的方式获得数据，所以不存在数据安全问题

(2)JS的模块化规范有哪些

  • CommonJS(ServerJS) ——-服务端应用广泛
  • ES6模块化 ——–浏览器端应用广泛

(3)导入与导出

​ 模块之间是隔离的，通过导入和导出进行数据和功能的共享

  • 导出**(暴露)**：模块公开其内部的一部分（如变量、函数等），使这些内容可以被其他模块使用
  • 导入(引入)：模块引入和使用其他模块导出的内容，以重用代码和功能

2. CommonJS的模块化规范

​ 双端都可以使用，但是一般使用在Node.js编写服务器

​ 注意不是在浏览器，this不指向window，它指向模块的exports对象

​ 面试题：注意：CommonJS的模块化规范导入的数据/方法是原先的复制品，与原先的js文件的内容没有半毛钱关系

​

​ 输出：

1

(1)CommonJS模块化语法

导入语法：
//在index.js中导入test.js模块
const obj = require('./test.js')
console.log(obj)
//当然可以在赋值后马上解构：const {name,getTel} = require('./test.js')
//注意：函数解构时不需要括号，调用时再加括号
//console.log(getTel())
//注意：如果解构的命名有冲突，则需要es6的加重命名
//const {name:schoolName,age} = require('./school.js')

​ 注意：如果test.js文件不主动暴露那么接受到的是一个空对象

导出(暴露)语法：（第一种方式）
//test.js
const name = '刘大业'

function getTel(){
	return '18899998888'
}

//私有不暴露
function getJob(){
	return '京东'
}

//暴露语法：
module.exports = {
	name:name  //es6新语法直接写name
	getTel  //es6新语法
}
导出(暴露)语法：（第二种方式）
//test.js
const name = '刘大业'

function getTel(){
	return '18899998888'
}

//私有不暴露
function getJob(){
	return '京东'
}

//暴露语法：
exports.name = name //这里是把name赋值给exports.name，在CommonJS模块化中this.name完全等价于exports.name
exports.getTel = getTel

(2)核心注意点

注意点1

​ 在CommonJS模块化中，会为每一个JS文件准备一个空对象。每个模块(JS文件)的内部的：this、exports、modules.exports在初始化时都指向这个空对象，该空对象就是当前模块导出的数据

​

注意点2

​ 无论如何修改导出对象，最终导出的都是module.exports指向的值，即如果你没有对 module.exports 重新赋值，使用 exports 或 module.exports 添加的属性都能成功导出，如果修改了就按module.exports指向的值导出

​ 范例1：面试题

//test.js的内容
exports = {a:1}
exports.b = 2
module.exports.c = 3
module.exports = {d:4}

//main.js导入test.js并输出对象的结果
const a = require('./test.js')
console.log(a)

//运行结果
//{ d:4 }

​ 图示：黄色框为默认的空对象，第一步把exports指向{a:1}，第二步将b=2添加到exports所指向的对象，第三步把c=3放入空对象，第四步将module.exports指向一个新对象{d:4}

​ 范例2：

​ 在没有修改 module.exports 的引用时，exports 和 module.exports 是同一个对象，因此通过 exports 和 module.exports 都可以向同一个对象添加属性。最终导出的是 module.exports 对象的内容

exports.name = 'John';
module.exports.age = 18;
//导出的是{name:'John',age:18}
注意点3

​ exports是对module.exports的初始引用(指向同一空对象)，仅为了方便给导出对象添加属性，所以不能使用exports = value 的形式导出数据，如果这样写会改变exports的指向（细节见上图）

(3)扩展理解：为什么模块之间是隔离的

​ 在CommonJS中，每一个写的JS文件的所有内容，在执行时已经被一个内置函数包起来了，所有写的代码都成为了这个函数的函数体，即形成了一个作用域

​ 内置函数如下：

//在test.js编写
...
console.log(arguments.callee.toString())
//在main.js引入test.js，并运行
//运行结果
function(exports,require,module,__filename,__dirname){
	/*test.js文件所有的内容*/
}

(4)Commonjs在浏览器运行

​ Node.js支持CommonJS但是浏览器不支持，直接导入到html页面运行会缺少参数require(见上面那个内置函数)，需要下载一个翻译官–browserify

安装browserify

  • 第一步：全局安装browserify：npm i browserify -g

  • 第二步：编译：

    //备注：main.js是源文件，build.js是输出的目标文件
browserify main.js -o build.js

  • 第三步：index.html页面中引入使用这个js

    <script type="text/javascript" src="./build.js"></script>

3. ES6的模块化规范(官方标准)

​ 几乎所有的前端项目，只要用了脚手架就是在使用ES6的模式在写，它是目前最流行的模块化规范，同时ES6也是支持浏览器端和服务端，但其默认不支持服务器

​ 面试题：注意：ES6的模块化规范导入的数据/方法是原先的公用一块内存，只读不可改，这个是和CommonJS不一样的地方

​

(1)ES6模块化语法

​ 注意：三种导出方式可以同时使用

导出语法：分别导出（第一种方式）

​ 谁要导出就在谁前面加export

//test.js
//导出属性
export const name = '刘大业'

//导出方法
export function getTel(){
	return '18899998888'
}

//私有不暴露
function getJob(){
	return '京东'
}
导出语法：统一导出（第二种方式）
//test.js
const name = '刘大业'

function getTel(){
	return '18899998888'
}

//私有不暴露
function getJob(){
	return '京东'
}

//统一导出：{}不是对象，是类似对象的方式，本质是一个标记
export{name,getTel} //因为不是对象不能写成name:name,getTel:getTel这种形式
导出语法：默认导出（第三种方式）
//test.js
const name = '刘大业'

function getTel(){
	return '18899998888'
}

//私有不暴露
function getJob(){
	return '京东'
}

//默认导出：只有一组值
//导出对象只有{default:900}
export default 900

/*
	//导出对象为{default:'刘大业'}
	export default name
*/

/*
	//导出多个的操作，导出对象为default:Object
	export default {
		name:name  //es6简写为name
		getTel:getTel //es6简写为getTel
	}
*/

​ 注意：命名导入和默认导入可以同时使用：import xyz,{name,getTel} from './test.js'

导入语法：万能语法，不管用什么导出方式都能接住（第一种方式）

​ 命名和分别导入、导出的值都一样，但是使用默认导出接到的值是default:obj，obj是一个对象，obj里面是默认导出的内容

//index.js
import * as test1 from './test.js'
console.log(test1)
//该文件需要放入浏览器中执行，所以需要引入到一个页面然后执行该页面
导入语法：命名导入，适用【分别导出、统一导出】（第二种方式）

​ 若命名冲突需要起别名如：import {name as SchoolName,getTel} from './test.js'

//index.js
import {name,getTel} from './test.js'
console.log(name)
console.log(getTel())
//该文件需要放入浏览器中执行，所以需要引入到一个页面然后执行该页面
导入语法：默认导入，适用【默认导出】（第三种方式）

​ 只要使用了默认导出，在导入时可以随便拿一个变量进行接收，接收的变量获得的是default:obj中的obj

//index.js
//qwe是随意起的名，适用默认导出
import qwe from './test.js'
console.log(qwe)  //运行结果：{900}
//该文件需要放入浏览器中执行，所以需要引入到一个页面然后执行该页面
导入语法：动态导入，不管用什么导出方式都能接住（第四种方式）

​ 在某一个特殊时间点：如点按钮、时间到了等，导入模块进行介入

//index.html
<body>
	<!-- 模块化导入语法 -->
	<script type="module" src="./index.js"></script>
	<button id="btn">导入模块</button>
</body>

//index.js
const btn = document.getElementById('btn')
//这里看不懂async和await的话blog里搜ES8特性
btn.onclick = async()=>{
	const result = await import('./student.js')
	console.log(result)
}
导入语法：import不接受任何数据（第五种方式）

​ 适用于每次启动就打印日志等行为

//log.js
console.log(Math.random())

//index.js
import './log.js'

(2)浏览器中运行ES6模块

​ 不写模块化导入语法浏览器无法正常执行

浏览器运行：需要写模块化导入语法
//index.html
<body>
	<!-- 传统导入js的方式 这里不行<script type="text/javasctipt" src=""></script> -- >
	
	<!-- 模块化导入语法 -->
	<script type="module" src="./index.js"></script>
</body>

​ 使用module关键字后，所有导出的字段在全局是搜索不到的，如果用text那么全局是搜索的到的

(3)Node(服务器)中运行ES6模块

​ 需要Node版本12及以上

第一种方法：把所有模块化的js文件改名为mjs

​ 最后运行那个导入了模块的mjs文件，如上面那个test.js和index.js都改名为test.mjs和index.mjs，用Node运行index.mjs

第二种方法：在工程文件目录增加一个package.json

​ 编写代码如下：

{
	"type":"module"
}
