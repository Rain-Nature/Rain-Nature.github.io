JS高级02-原型

JS高级
Publish Date:   2025-01-04

1. 深入面向对象

​ JS通过构造函数和原型来实现面向对象

​ 前端面向过程更多，但也有面向对象

2. 原型

(1)案例引入

function Star(){
	this.sing = function(){
		console.log('我会唱歌')
	}
}
const ldy = new Star()
const zxy = new Star()
//obj.sing 没有括号表示函数本身，加括号表示要调用这个函数方法
console.log(ldy.sing === zxy.sing)  //false

​ 因为在构造函数中声明函数时，每次new一个新对象，都会再次为这个函数分配空间，这会造成内存的浪费。而通过原型可以解决这个问题

(2)原型对象(prototype)

​ 公共的属性写在构造函数身上，公共的方法写在原型身上

原型对象的目的是实现方法的共享
  • 构造函数通过原型分配的函数是所有对象共享的
  • JS规定，每一个构造函数都有一个prototype属性，指向另一个对象，我们也称为原型对象
  • 原型对象可以挂载函数，对象实例化不会多次创建原型上的函数
  • 我们可以把不变的方法，直接定义在prototype对象上，这样所有对象的实例就可以共享这些方法了
  • 构造函数和原型对象中的this都指向调用的实例化对象，需要注意的是实例化对象要调用一下原型里的方法，原型对象的this才指向实例对象，而构造函数的this需要new一个对象才指向实例对象，否则都指向全局对象 (window)
原型的使用
function Star(){

}
Star.prototype.sing = function(){
	console.log('我会唱歌')
}
const ldy = new Star()
const zxy = new Star()
console.log(ldy.sing === zxy.sing)  //true

(3)constructor属性

​ 每一个原型对象里默认都有constructor属性，作用是指向该原型对象的构造函数

console.log(Star.prototype.constructor === Star) //true

constructor的使用：指回原型对象的构造函数
function star(){

}

//使用prototype一共有两种情况

//第一种情况：直接给prototype追加方法
Star.prototype.dance = function(){
	console.log('跳舞')
}
Star.prototype.sing = function(){
	console.log('唱歌')
}

//第二种情况：以对象的形式，直接赋值给Star.prototype
Star.prototype = {  //赋值，整个替换
	//重新指回这个原型对象的构造函数，不写的话对象中的constructor属性就丢了
	constructor:Star,
	//函数的对象里声明
	dance:function(){
		console.log('跳舞')
	},
	sing:function(){
		console.log('唱歌')
	}
}

(4)对象原型(__proto__)

​ 在每一个实例对象中都会有一个属性__proto__指向构造函数的prototype对象，由此实例对象可以直接使用原型的方法

注意事项

  • [[prototype]]和__proto__意义相同，浏览器显示[[prototype]]，写代码写__proto__

  • __proto__是一个只读属性不可以进行修改

  • __proto__中也有一个constructor属性，指向创建该实例对象的构造函数

    console.log(ldy.__proto__ === Star.prototype)  //true
console.log(ldy.__proto__.constructor === Star)   //true

(5)JS的继承：原型继承

​ JS的面向对象大多借助原型对象实现继承的特性

​ 核心思想：子类构造函数的原型= new 父类构造函数

使用原型对象来实现继承(错例)

​ 样例：这个居然用一个写死的对象来当被继承的，我刚看还没反应过来，也是真的逆天

//父类Person，这里是一个对象，而不是构造函数
const Person = {
	head:1,
	eyes:2
}

//子类女人 构造函数 需要继承Person
const Woman = {

}

//核心步骤1：Women通过原型对象来继承Person，这是覆盖操作
Woman.prototype = Person

//核心步骤2：需要将Woman.prototype指回Women构造函数，不写这个指不回原来的Woman了
Woman.prototype.constructor = Woman

//生成Women的实例
const red = new Woman()

//打印父类的属性
console.log(red.head)  //1
上面代码的问题

​ obj.prototype都是在堆里生成，如果直接等于Person，那么指向同一个地址，那么对于Woman.prototype的修改（增加函数），Man.prototype也会获得该函数，这是有问题的

问题解决(正例)：与错的对比着看

​ 使用构造函数，通过new关键字来生成不同的对象来解决上面指向同一地址的问题

//构造函数
function Person(){
	this.eyes = 2
	this.head = 1
}

//女人 构造函数
function Woman(){

}

//通过原型继承Person
Woman.prototype = new Person()
//指回Woman
Woman.prototype.constructor = Woman

//给女人添加一个方法
Women.prototype.baby = function(){
	console.log('宝贝')
}

const red = new Woman()
console.log(red)

(6)原型链

​ 基于原型对象的继承使得不同构造函数的原型对象关联在一起，并且这种关联的关系是链状结构，我们将原型对象的链状结构关系称为原型链

​ 注意：只要是对象就有__proto__指向原型对象，只要是原型对象就有constructor指向创造它的构造函数

原型链的原理

​ 原型对象中也有一个__proto__的属性，它的指向如下

​ JS中有一个最大的类Object，普通类的构造函数的原型对象里的对象原型指向Object的原型对象

console.log(Person.prototype.__prot__ === Object.prototype)   //true

​ 而Object.prototype.__proto__指向null

console.log(Object.prototype.__proto__)   //null
原型链图示

原型链的使用

​ 因为原型对象里的方法，实例可以使用，那么通过继承，则可以获得父类的方法

​ 原型链-查找规则图示：

​ 第6条样例：

console.log(red instanceof Person)   //true
console.log(red instanceof Object)   //true
console.log(red instanceof Array)   //false
console.log([1,2,3] instanceof Array)   //true
console.log(Array instanceof Object)   //true

3. 深浅拷贝

​ 浅拷贝、深拷贝只针对引用类型

​ 因为如果对引用类型采用直接赋值的操作进行拷贝，老对象和新对象在堆中指向的同一个对象，对新对象修改就是在对老对象进行修改，两个对象相互影响这样不符合拷贝的初衷

(1)浅拷贝：

​ 对引用数据执行浅拷贝，它里面是简单数据类型则直接拷贝值，如果是引用数据类型则拷贝的是地址

常见方法：

​ 1.拷贝对象：Object.assign() / 展开运算符{…obj}

​ 2.拷贝数组：Array.prototype.concat() / […arr]

对象拷贝的例子：
const obj = {
	uname:'pink',
	age:18
}
//浅拷贝
const o = {...obj}
//或 
//const o = {}
//Object.assign(o,obj)
o.age = 20
console.log(o)   //20
console.log(obj)   //18

​ 数组自己举一反三

​ 浅拷贝只拷贝最外面这一层，如果对象里还有对象（即引用数据类型），那对象的对象修改，又会出现直接赋值的情况
