1. NoSQL数据库

2. Redis的安装

​ Redis是使用C写的程序，目前主流在Linux操作系统，官方不提供Window版，最新的Redis版：Redis7.x

官网下载：https://redis.io/download/

(1)安装步骤(社区版)

​ 1.切换到空白目录

​ 2.下载Redis源码的压缩包

​ 3.下载gcc，用于编译c源码

​ 4.解压Redis源码

​ 5.进入解压后的Redis文件夹

​ 6.编译源码

​ 7.安装Redis

​ 8.默认安装在/usr/local/bin，可以切换过去查看

cd /usr/local/src
sudo curl -O https://download.redis.io/releases/redis-6.2.1.tar.gz
apt install gcc
tar -zxvf redis-6.2.1.tar.gz
cd redis-6.2.1
make
make install
cd /usr/local/bin

(2)Redis的全部文件的作用（目录usr/local/bin）

​ redis-benchmark：性能测试工具，可用于本地运行，查看 Redis 性能

​ redis-check-aof：修复有问题的 AOF（Append-Only File）日志文件

​ redis-check-dump：修复有问题的 dump.rdb 数据库文件

​ redis-sentinel：Redis 集群的管理工具

​ redis-server：Redis 服务器的启动命令

​ redis-cli：Redis 客户端，提供命令行操作入口55

(3)Redis的启动（默认端口：6379）

​ 有前台启动和后台启动

Redis的前台启动（不推荐）

​ 输入以下命令启动，注意：命令行窗口不能关闭，否则服务器停止

//要在redis-server所在的目录内执行
redis-server
Redis的后台启动（推荐）

​ 进入到之前redis源码目录中：/usr/local/src/redis-6.2.1

​ 将redis.conf的文件复制到etc目录（这个自便）：cp redis.conf /etc/redis.conf

​ 将复制后的conf文件里的daemonize由no修改为yes：cd /etc

​ 回到/usr/local/bin（redis-server所在的目录），启动Redis：redis-server /etc/redis.conf

​ 启动后输入：redis-cli，可以通过客户端连接上redis

​ 关闭方式：shutdown

3. Redis的五大数据类型

​ Redis对所有数据的操作都是原子性的，是单线程不会被打断的，但是批量操作有一个失败的话，全部操作都会失败，每一句代码叫做一个操作，msetnx就属于批量操作

(1)Redis数据库的结构

​ Redis默认16个数据库，初始默认使用0号库

​ 使用select <num>来切换数据库，如：select 8

​ 统一的密码管理，所有的库都是用同样的密码

(2)Redis中对Key的操作

命令 作用
keys * 查看当前库所有 key
exists <key名> 判断某个 key 是否存在
type <key名> 查看 key 的数据类型
del <key名> 删除指定 key（key 和 value 都删）
unlink <key名> 非阻塞删除 key（异步删除数据）

(3)String（字符串）

​ String是Redis里面最基本的类型，一个key对应一个value

​ String类型是二进制安全的，意味着Redis可以包含任何数据，比如jpg图片或序列化对象

​ String类型的底层实现类似于Java的ArrayList，采用预分配冗余空间的方式来减少内存的频繁分配，最大长度为512M（仅Value，不包含key）

String类型的常用命令
命令 作用 说明
set <key> <value> [NX] [XX] [EX] [PX] 添加键值对 NX：key 不存在时添加；XX：key 存在时修改；EX：设置过期时间（秒）；PX：设置过期时间（毫秒）
get <key> 获取 key 对应的 value 无
append <key> <value> 在原 value 末尾追加新内容 无
incr <key> 将 key 对应的数值 +1（value 必须是数字） 无
decr <key> 将 key 对应的数值 -1（value 必须是数字） 无
String类型的适用场景
应用场景 示例命令与描述 说明
计数器 incr view_count 记录网站访问量、视频播放量
共享 Session set session:12345 “user_data” ex 3600 分布式系统存储用户 Session
分布式锁 setnx lock:order:1001 “locked” 控制并发，防止超卖
限流 incr user:1001:requests 记录 API 调用次数，做限流
配置存储 set system:config ‘{“theme”:”dark”}’ 存储全局配置，如系统设置

(4)List（列表）

​ List类型属于单key多值，值又全部存储在value中

​ List类型的底层是个双向链表，两端操作性高，中间操作性差；当列表元素较少时会使用一块连续的内存存储，叫zipset，当元素变多时将多个zipset双向链接，叫quicklist

List类型的常用命令
命令 作用 说明
lpush <key> <value1> <value2> ... 从左侧插入一个或多个值 队列结构，先进后出
rpush <key> <value1> <value2> ... 从右侧插入一个或多个值 队列结构，先进先出
lrange <key> <start> <stop> 按索引范围获取列表元素 0 -1 代表获取全部
lpop <key> 从左侧弹出一个值 若列表为空，key 自动销毁
rpop <key> 从右侧弹出一个值 若列表为空，key 自动销毁
List类型的适用场景
应用场景 相关命令 典型案例
消息队列 lpush rpop 任务队列、秒杀下单
文章/评论列表 lpush lrange 博客评论、微博动态
延迟队列 blpop brpop 异步任务、订单处理

(5)Set类型（集合）

​ Set类型的功能和List类似，区别是set可以自动排重

​ Set 类型的底层数据结构是基于哈希表实现的（也可以理解为字典），它通过哈希表实现了元素的添加、删除、查找操作，时间复杂度为 O(1)，与 Java 中的 HashSet 类似。

set类型的常用命令

sadd key members [....] 将一个或多个 members 添加到 key 集合 集合中已存在的元素不会重复添加
smembers key 获取 key 集合中的所有元素 以列表形式返回集合中的所有值
srem key members [....] 从 key 集合中删除一个或多个 members 仅删除匹配的成员，不影响其他成员
spop key count 随机从 key 集合中取出 count 个元素 取出的元素会从集合中移除
set类型的应用场景
应用场景 相关命令 典型案例
防止重复 sadd sismember scard 防止刷票、签到去重
随机抽奖 srandmember spop 抽奖、随机推荐

(8)新数据类型(Bitmaps、HyperLogLog、Geospatial)

Bitmaps

​ 针对位操作的数据类型，本质还是String的key、value，只是它一个字节可以存8个数据，即一位一个数据，只存false和true，适用于统计用户是否上线

HyperLogLog

​ 针对基数运算，就是统计存入的值是否重复，重复不存入，它是使用算法推算值是否重复，值本身不存储，但该算法有概率失败，适用与统计用户博客访问次数

Geospatial

​ 针对地理信息经纬度操作，存值经纬度，其可以通过两个经纬度算出之间距离、某范围查满足城市的信息等实用操作

​ 使用Redis发布信息时可以指定要发送到的频道，同时Redis客户端可以订阅任意数量的频道

(2)Jedis案例-手机验证码

案例要求

​ 1.输入手机号，点击发送后随机生成6位数字码，2分钟内有效

​ 2.输入验证码，点击验证，返回成功或失败

​ 3.每个手机号每天只能输入3次

案例实现

​ 1.静态方法：生成一个6位的验证码

//生成6位数字验证码
public static String getCode() {
	Random rand = new Random();
	String code = "";
	for (int i = 0; i < 6; i++) {
		code += rand.nextInt(10);
	}
	return code + "";
}

​ 2.静态方法：校验手机的次数key，将手机号及对应的验证码放到redis中，设置过期时间

public static void verifyCode(String phone, String code) {
    Jedis jedis = new Jedis("192.168.199.129", 6379);
    //字符串拼接：手机发送次数的key
    String countKey = "VerifyCode:" + phone + ":count";
    //字符串拼接：验证码的key
    String verifyCode = "VerifyCode:" + phone + ":code";

    String count = jedis.get(countKey);
    if (count == null) {
        jedis.setex(countKey, 24*60*60,"1");
    }else if (Integer.parseInt(count) < 3) {
            jedis.setex(countKey, 24*60*60,Integer.parseInt(count)+1+"");
    }else {
        System.out.println("请明日再试");
        jedis.close();
        return;
    }

    //验证码放入redis，设置过期时间为2分钟
    jedis.setex(verifyCode,120, code);
    jedis.close();
}

​ 3.静态方法：验证码校验

public static void verifyCodeText(String phone,String code) {
    Jedis jedis = new Jedis("192.168.199.129", 6379);
    String verifyCode = "VerifyCode:" + phone + ":code";
    String jCode = jedis.get(verifyCode);
    if (jCode.equals(code)) {
        System.out.println("登录成功");
    }else {
        System.out.println("登录失败");
    }
    jedis.close();
}

​ 3.创建配置类RedisConfig.java（这个是固定的，不用自己写）

//RedisConfig.java

@Configuration
@EnableCaching
public class RedisConfig extends CachingConfigurerSupport {

    @Bean
	public RedisTemplate<String, Object> redisTemplate(RedisConnectionFactory redisConnectionFactory) {
        RedisTemplate<String, Object> template = new RedisTemplate<>();
        RedisSerializer<String> redisSerializer = new StringRedisSerializer();
        ObjectMapper om = new ObjectMapper();
        om.setVisibility(PropertyAccessor.ALL, JsonAutoDetect.Visibility.ANY);
        om.activateDefaultTyping(
                BasicPolymorphicTypeValidator.builder()
                        .allowIfBaseType(Object.class)
                        .build(),
                ObjectMapper.DefaultTyping.NON_FINAL
        );

        Jackson2JsonRedisSerializer<Object> jackson2JsonRedisSerializer =
                new Jackson2JsonRedisSerializer<>(om, Object.class);

        template.setConnectionFactory(redisConnectionFactory);
        template.setKeySerializer(redisSerializer);
        template.setValueSerializer(jackson2JsonRedisSerializer);
        template.setHashKeySerializer(redisSerializer);
        template.setHashValueSerializer(jackson2JsonRedisSerializer);
        template.afterPropertiesSet();

        return template;
    }

    @Bean
    public CacheManager cacheManager(RedisConnectionFactory factory) {
        RedisSerializer<String> redisSerializer = new StringRedisSerializer();

        ObjectMapper om = new ObjectMapper();
        om.setVisibility(PropertyAccessor.ALL, JsonAutoDetect.Visibility.ANY);
        om.activateDefaultTyping(
                BasicPolymorphicTypeValidator.builder()
                        .allowIfBaseType(Object.class)
                        .build(),
                ObjectMapper.DefaultTyping.NON_FINAL
        );

        Jackson2JsonRedisSerializer<Object> jackson2JsonRedisSerializer =
                new Jackson2JsonRedisSerializer<>(om, Object.class);

        RedisCacheConfiguration config = RedisCacheConfiguration.defaultCacheConfig()
                .entryTtl(Duration.ofSeconds(600)) // 默认缓存时间
                .serializeKeysWith(RedisSerializationContext.SerializationPair.fromSerializer(redisSerializer))
                .serializeValuesWith(RedisSerializationContext.SerializationPair.fromSerializer(jackson2JsonRedisSerializer))
                .disableCachingNullValues();

        return RedisCacheManager.builder(factory)
                .cacheDefaults(config)
                .build();
    }
}

(4)RedisTemplate的序列化问题

问题

​ 在Redis中的key和value均是以二进制的形式存储的，因此客户端输入的key和value都会经过序列化之后才发往Redis服务端。而RedisTemplate所使用序列化方式和命令行客户端采用序列化方式不相同，进而导致序列化之后的二进制数据不同，此时就无法互通，即RedisTemplate存入的数据，在Redis客户端中无法正确读取

解决

​ 通过定义配置类，设置RedisTemplate的序列化方式解决以上问题，偷懒可以使用StringRedisTemplate，直接注入使用即可

RedisTemplate 和 StringRedisTemplate 的序列化器区别：
  • RedisTemplate<Object, Object> 默认使用的是 JDK 序列化器，可以存任意类型的对象，但存入的数据是二进制格式，Redis客户端中（使用redis-cli等）不可读，但程序不影响
  • StringRedisTemplate 是 RedisTemplate<String, String> 的封装，专门为操作字符串类型的 key 和 value 而设计，使用的是 StringRedisSerializer，存入的数据是纯字符串格式，Redis客户端中（使用redis-cli等）可读
