Linux03-Linux系统高级配置及相关命令

Linux
Publish Date:   2025-05-18

1. Linux终端的快捷操作

(1) 强制停止（ctrl+c）

  • 用于强制停止某些卡住的命令
  • 退出当前命令的输入

(2) 退出或登出（ctrl+d）

  • 退出账户的登录(如：root用户)
  • 退出编译器(注意：不能退出vi/vim)

(3) 光标移动快捷键

  • ctrl+a，跳到命令开头
  • ctrl+e，跳到命令结尾
  • ctrl+键盘左键，向左跳一个单词
  • ctrl+键盘右键，向右跳一个单词

(4) 清屏

  • 通过快捷键ctrl+l，可以清空终端内容
  • 通过命令clear也可以得到同样效果

2. Linux的软件安装

​ 在软件安装中CentOS系统和Ubuntu是使用不同的包管理器

(1)CentOS的软件安装

​ yum：PRM包软件管理器，用于自动化安装配置Linux软件，并可以自动解决依赖问题，类似于Windows的微软商店

​ 使用yum命令可以在CentOS的Linux环境下安装程序

​ 语法：yum [-y] [install | remove | search] 软件名称

  • -y：自动确认，无需手动确认安装或卸载过程
  • install：安装
  • remove：卸载
  • search：搜索网上有无这个软件

​ 注意：

  1. yum命令需要root权限，可以su切换到root，或使用sudo提取
  2. yum命令需要联网

(2)Ubuntu的软件安装

​ 使用apt命令可以在Ubuntu的Linux环境下安装程序

​ 语法：apt [-y] [install | remove | search] 软件名称

​ 用法和yum一致，注意事项也一致

3. Linux中服务的启动和关闭

​ Linux系统很多软件(内置或第三方)均支持使用systemctl命令控制：启动、停止、开机自启

​ 能够被systemctl管理的软件，一般被称为：服务

​ 语法：systemctl start | stop | status | enable | disable 服务名(内置、第三方)

  • start：启动
  • stop：关闭
  • status：查看状态
  • enable：开启开机自启
  • disable：关闭开机自启
部分系统内置的服务(名)：
  • NetworkManager：主网络服务
  • network：副网络服务
  • firewalld：防火墙服务
  • sshd，ssh服务：FinalShell远程登录Linux使用的服务

​ 注意：部分软件安装后没有自动集成到systemctl中，需要手动添加

4. IP地址和主机名

(1)IP地址

​ 每一台联网的电脑都会有一个地址，用于和其他计算机进行通讯

​ IP地址主要有2个版本，V4版本和V6版本(V6很少用)

​ IPv4版本的地址格式是：a.b.c.d，其中abcd表示0~255的数字，如192.168.88.101就是一个标准的IP地址

​ 可以通过命令：ifconfig，查看本机的ip地址，如无法使用ifconfig命令，可以安装：yum -y install net-tools

特殊IP地址
  • 127.0.0.1，这个IP地址用于指代本机
  • 0.0.0.0
    • 可以用于指代本机
    • 可以在端口绑定中用来确定绑定关系
    • 在一些IP地址限制中，表示所有IP的意思，如放行规则设置为0.0.0.0，表示允许任意IP访问

​ Ubuntu的系统可能不会自动获取IP地址，使用以下命令手动获取

sudo dhclient ens33

(2)主机名

​ 每一台电脑除了对外联络地址(IP地址)以外，也可以有一个名字，称为主机名

​ 无论是Windows或Linux系统，都可以给系统设置主机名

在Linux中修改主机名
  • 查看主机名使用命令：hostname
  • 修改主机名使用命令：hostnamectl set-hostname 新主机名

    其中ens33是目前的以太网接口，故其配置文件为vim /etc/sysconfig/network-scripts/ifcfg-ens33。

  2. 修改网络配置文件

    打开配置文件

    [root@localhost ~]# vim /etc/sysconfig/network-scripts/ifcfg-ens33

    内容如下

    TYPE=Ethernet
PROXY_METHOD=none
BROWSER_ONLY=no
BOOTPROTO=dhcp
DEFROUTE=yes
IPV4_FAILURE_FATAL=no
IPV6INIT=yes
IPV6_AUTOCONF=yes
IPV6_DEFROUTE=yes
IPV6_FAILURE_FATAL=no
IPV6_ADDR_GEN_MODE=stable-privacy
NAME=ens33
UUID=bf788989-e095-4e58-865a-e6b07248a514
DEVICE=ens33
ONBOOT=yes

    需要修改的内容如下

    • 修改BOOTPROTO参数为static

    • 修改ONBOOT参数为yes

    • 增加如下内容

      # 子网IP
IPADDR=192.168.10.100
# 子网掩码
NETMASK=255.255.255.0
# 网关IP
GATEWAY=192.168.10.2
# DNS在虚拟机中由网关代为转发
DNS1=192.168.10.2

      注意事项

      IPADDR需要位于VMnet8子网IP范围内，GATEWAY需要配置为的NAT服务的网关IP。

  3. 重新启动网络服务

    在终端执行以下命令，重启network服务

    [root@localhost ~]# systemctl restart network

  4. 观察网络配置是否生效

    在终端执行ifoconfig命令，观察ens33的网络接口的IP地址是否为前文所配置的IPADDR。

5. 服务器连接和文件下载

(1)Linux测试服务器连接

​ 通过ping命令可以检查指定的网络服务器是否是可联通状态

​ 语法：ping [-c num] ip或主机名

  • -c：检查的次数，一般不使用-c选项，将无限次数持续检查
  • ip或主机名，被检查的服务器的ip地址或主机名地址

(2)文件下载

wget实现文件下载（更适合下载文件（尤其是大文件或网站镜像））

​ wget是非交互式的文件下载器

​ 通过wget命令可以在命令行内下载网络文件

​ 语法：wget [-b] url

  • -b：后台下载，会将日志写入到当前工作目录的wget-log文件

  • url：下载链接

    注意：无论下载是否完成，都会生成要下载的文件，如果下载未完成，请及时清理未完成的不可用文件

curl实现文件下载（更适合调试 HTTP 接口、处理复杂请求）

​ 通过curl命令可以发送http网络请求(GET、POST等)，可用于：下载(小)文件、获取信息(网页源代码)等

​ 语法：curl [-o] url

  • -o：用于下载文件，当url是下载链接时，可以使用此选项保存文件
  • url：要发起请求的网络地址

6. 端口与进程管理

(1)端口

​ 端口是设备与外界通讯交流的出入口。端口可以分为：物理端口和虚拟端口两类

  • 物理端口：又称为接口，是可见的端口，如USB、HDMI等
  • 虚拟端口：指计算机内部的端口，是不可见的，是用来操作系统和外部进行交互使用的
计算机程序之间的通讯原理

​ 通过IP只能锁定计算机，无法锁定具体的程序

​ 通过端口可以锁定计算机上具体的程序，确保程序之间进行沟通

​ IP地址相当于小区地址，在小区内可以有许多住户（程序）而门牌号（端口）就是各个住户（程序）的联系地址

Linux系统的端口范围

​ Linux系统是一个超大号小区，可以支持65535个端口，这6万多个端口分为3类进行使用

  • 公认端口：0~1023：通常用于一些系统内置或知名程序的预留使用，如SSH服务的22端口，HTTPS服务的443端口，非特殊需要，不要占用这个范围的端口
  • 注册端口：1024~49151：通常可以随意使用，用于松散的绑定一些程序\服务
  • 动态端口：49151~65535：通常不会固定绑定程序，而是当程序对外进行网络链接时(即出口)，用于临时使用

端口号的查看

​ 通过nmap命令可以查看某台机器的哪些端口开着

​ 语法：nmap IP地址

​ 注意：nmap命令不是系统自带的需要安装：yum -y install nmap

​ 通过netstat命令（监听本地机器）可以排查端口被哪个进程占用、本地服务是否监听

​ 语法：netstat -anp | grep 端口(进程)号

​ 注意：netstat命令不是系统自带的需要安装：yum -y install net-tools

​ 区别

特性 nmap netstat
作用范围 扫描“别的机器”或本机的开放端口 查看“本机”当前网络连接和端口占用情况
原理 通过发送网络探测包判断目标 IP 的开放端口（外部扫描） 查看本地系统内核维护的网络连接表（内部查看）
用途 网络安全扫描、端口探测 排查端口占用、网络连接、查看服务监听情况
权限要求 非 root 只能做基础扫描，root 权限下功能更强 一般也需要 root 才能查看全部进程
是否显示进程名 不能直接看到服务名和进程（仅端口） 能看到进程 PID 和程序名（加 -p 参数）

(2)进程管理

​ 程序运行在操作系统中是被操作系统管理的

​ 为管理运行的程序，每个程序在运行的时候，便被操作系统注册为系统中的一个：进程，并会为每一个进程都分配一个独有的：进程ID

查看进程

​ 通过ps命令可以查看Linux系统中的进程信息

​ 语法：ps [-e -f]

  • -e：显示出全部的进程

  • -f：以完全格式化的形式展示信息(展示全部信息)

    固定用法为：ps -ef，列出全部进程的全部信息

​ 全部进程的全部信息如下图所示：

​ 上图从左到右依次为：

  • UID：进程所属的用户ID
  • PID：进程的进程号ID
  • PPID：进程的父ID(启动此进程的其他进程)
  • C：此进程的CPU占用率(百分比)
  • STIME：进程的启动时间
  • TTY：启动此进程的终端序号，如显示? ，表示非终端启动
  • TIME：进程占用CPU的时间
  • CMD：进程对应的名称或启动路径或启动命令
ps的使用技巧

​ 配合grep可以查看想要查看的进程，例如

//查看所有的Java进程
ps -ef | grep java
终止进程

​ 在Windows系统中，可以通过任务管理器选择进程，点击结束进程来关闭它

​ 在Linux系统中，通过kill命令来终止进程

​ 语法：kill [-9] 进程ID

  • -9：表示强制终止进程，不使用选项会向进程发送信号要求其关闭，但是否关闭要看进程自身的处理机制

7. 主机状态

(1)查看系统资源占用

​ 通过top命令查看CPU、内存使用情况，类似Windows系统的任务管理器

​ 语法：top

​ 注意：界面默认每5秒刷新一次，按q或ctrl+c退出

​

(2)磁盘信息监控

​ 通过df命令可以查看硬盘的使用情况

​ 语法：df [-h]

  • -h，以更加人性化的单位显示

​ 通过iostat命令可以查看CPU、磁盘的相关信息

​ 语法：iostat [-x] [num1] [num2]

  • -x，显示更多信息
  • num1，数字，刷新间隔
  • num2，数字，刷新几次

(3)网络状态监控

​ 通过sar命令可以查看网络的相关统计(sar命令非常复杂，这里仅简单用于统计网络)

​ 语法：sar -n DEV [num1] [num2]

  • -n，查看网络
  • DEV，查看网络接口
  • num1，刷新间隔(不填就查看一次结束)
  • num2，查看次数(不填无限次数)

8. 环境变量

​ 通过env命令可以查看当前系统中记录的环境变量

​ 语法：env

​ 注意：环境变量是一种KeyValue结构

​ 使用echo ${Key} 可以直接在命令行输出Value，即${Key} 可以取到环境变量的值

Linux环境变量的自行设置

  • 临时设置，语法：export 变量名=变量值

  • 永久生效

    • 针对当前用户生效，配置在当前用户的：vi ~/.bashrc在文件中加export 变量名=变量值；保存后通过语法：source .bashrc生效

    • 针对所有用户生效，配置在系统的：vim /etc/profile 文件中加export 变量名=变量值；保存后通过语法：source /etc/profile生效

9. Linux文件的压缩与解压

(1)常用格式.tar和.gz

​ tar就是打包，gz才是压缩

  • .tar，称为tarball，归档文件，即简单的将文件组装到一个.tar的文件内，并没有太多文件体积的减少，仅仅是简单的封装
  • .gz，常见的为.tar.gz，gzip压缩文件，即使用gzip压缩算法将文件压缩到一个文件内，可以极大的减少压缩后的体积
书本语法

​ 针对上述两种格式，使用tar命令均可以进行压缩和解压缩的操作

​ 打包：tar [-c -f] 包名 文件1 文件2 ... 文件N

​ 解包：tar [-x -f] 包名 -C 指定目录

  • -c，创建压缩文件，用于压缩模式
  • -v，显示压缩、解压过程，用于查看进度
  • -x，解压模式
  • -f，要创建的文件，或要解压的文件，-f选项必须在所有选项中位置处于最后一个
  • -z，gzip模式。不使用-z就是普通的tarball格式，**-z选项一般使用时放在选项位第一个**
  • -C 指定目录，选择解压的目的地，用于解压模式，**-C要单独在后面列出，和解压所需的其他参数分开**
样例

  1. 压缩的例子：将1.txt，2.txt，3.txt压缩到test.tar文件内

    tar -cvf test.tar 1.txt 2.txt 3.txt

  2. 解压的例子

    • 解压test.tar

      tar -xvf test.tar

    • 解压test.tar到指定目录(/home/rain)

      tar -xvf test.tar -C /home/rain/

(2)ZIP格式的压缩和解压

书本语法
  • 压缩的语法：zip [-r] xxx.zip 要压缩的文件路径
    • -r，参数中有文件夹时使用
  • 解压的语法：unzip xxx.zip [-d 目录]
    • -d，指定解压去的目录。用法和上面的-C一样
样例

  1. 压缩的例子：将file1.txt和file2.txt，压缩为first.zip

    zip first.zip file1.txt file2.txt

    1. 解压缩的例子：将first.zip解压到zip目录

      mkdir zip
unzip first.zip -d zip/
