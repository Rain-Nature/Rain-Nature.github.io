子网与子网掩码和DNS

计算机网络
Publish Date:   2025-05-17

1. 子网（Subnet）

​ 子网是大型网络被划分成多个较小网络的结果。划分子网的目的是为了更高效地利用 IP 地址和更灵活地进行网络管理。

​ 在私有网络（内网）中，子网 IP 是可以自己定的

一个子网就是一个“IP 地址范围”，属于同一个“网络地址”

2. 子网掩码（Subnet Mask）

​ 子网掩码的作用是标识IP 地址中哪些是网络位，哪些是主机位

​ 子网掩码将一个 IP 地址分解成两部分：

IP地址 = 网络地址 + 主机地址

​ 子网掩码与子网IP按位与之后得出网络地址，比较网络地址是否相等即可判断是否同一子网。

特点：
  • 32 位的二进制数
  • 连续的 1 表示网络位，连续的 0 表示主机位
  • 常见表示形式：
    • 点分十进制：255.255.255.0
    • CIDR 表示法：/24，表示前 24 位是网络位
子网掩码的使用：
  • 计算某个 IP 所在的网络地址
  • 判断两个 IP 是否在同一子网
  • 划分出多个子网段，方便管理和路由转发

3. 子网划分的经典例子

(1)题目：

​ 将 192.168.1.0/24 划分为 4 个子网，问每个子网的范围是多少？192.168.1.62 属于哪个子网？

步骤 1：分析子网数量
  • 原始：/24 → 主机位 8 位，可容纳 2⁸ = 256 个地址
  • 目标：分成 4 个子网 → 需要多用 2 位作为网络位（2² = 4）
  • 新掩码：/26 → 主机位变为 6 位 → 每个子网有 2⁶ = 64 个地址
步骤 2：列出子网范围（每个步长为 64）
子网编号 网络地址 可用主机范围 广播地址
子网 1 192.168.1.0 192.168.1.1 ~ 62 192.168.1.63
子网 2 192.168.1.64 192.168.1.65 ~ 126 192.168.1.127
子网 3 192.168.1.128 192.168.1.129 ~ 190 192.168.1.191
子网 4 192.168.1.192 192.168.1.193 ~ 254 192.168.1.255
步骤 3：判断 IP 是否属于某个子网

查找 192.168.1.62 属于哪个子网？

  • 它在范围 192.168.1.1 ~ 62 中
  • ✅ 所以它属于子网 1，即 192.168.1.0/26

(2)用二进制演示子网划分（按位与）

IP1：192.168.1.62

子网掩码：255.255.255.192（即 /26）

转成二进制：

IP       : 11000000.10101000.00000001.00111110
子网掩码 : 11111111.11111111.11111111.11000000
按位与结果:
         = 11000000.10101000.00000001.00000000
         = 192.168.1.0 （这是网络地址）

再看另一个 IP，例如：192.168.1.100：

IP       : 11000000.10101000.00000001.01100100
掩码     : 11111111.11111111.11111111.11000000
结果     : 11000000.10101000.00000001.01000000
         = 192.168.1.64
判断是否同一网段的口诀
IP1 & 掩码 == IP2 & 掩码 → 同网段

4. DNS 与 hosts 区别

项目 hosts（本地） DNS（网络）
本质 本地静态 IP 域名映射文件 分布式域名解析系统
作用 手动指定 IP 自动解析域名
优先级 高，系统先查 hosts 后查 DNS
控制权 本地用户控制 域名所有者控制

你可以把 DNS 理解为“云端大电话簿”，hosts 是你“本地私人通讯录”
